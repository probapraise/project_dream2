# Community Grammar — Layer A

## 정의

레이어 A는 **시뮬레이션 창발 레이어**다.
P-* 슬롯의 빅5/파생지표 조건 기반 상호작용이 누적되면서, 작가가 의도하지 않은 문화·규범이 자생하는 계층.

문화는 사전에 정의되지 않는다. 시뮬레이션 결과에서 귀납적으로 발견되고, 작가 승인 후 다음 시뮬레이션의 컨텍스트로 환류된다.

---

## 적용 범위

- **대상**: 각인광장 내 모든 게시판
- **제외**: BOARD-001 낙서장 (Layer B 전용 공간)
- **게시판 수**: 커뮤니티 맵 확장에 따라 동적으로 증가

---

## 핵심 철학

> 문화는 규칙이 아니라 중력이다.

- 등재된 문화는 캐릭터에게 "이렇게 행동하라"는 명령이 아니다.
- "이 공간에서는 이런 경향이 지배적이다"는 확률적 맥락으로 주입된다.
- 캐릭터는 자신의 빅5/파생지표에 따라 문화를 흡수하거나, 저항하거나, 충돌을 일으킨다.
- 충분한 이탈과 충돌이 누적되면 문화 자체가 변동될 수 있다.

## 운영 파이프라인

```
[Cold Start — 첫 시뮬레이션]
문화 바이블 없이 실행
  → 순수하게 P-* 빅5/파생지표만으로 행동 생성
  → 어떤 문화도 사전 주입하지 않음

        ↓ 라운드 종료 후

[Post-sim Synthesis]
오케스트레이터: 시뮬 결과 종합
  → "이 게시판에서 이런 패턴이 관찰됩니다" 보고
  → 작가와 문화 리뷰 대화
  → 작가 승인 시 → CULTURE 레코드 등재

        ↓ 다음 시뮬레이션부터

[문화 주입]
등재된 CULTURE 레코드를 시뮬 컨텍스트에 "경향"으로 주입
  → 대다수 캐릭터: 자연스럽게 문화 흡수·재현
  → 일부 캐릭터: 은근히 저항하거나 무시
  → 소수 캐릭터: 정면 충돌 → 담론화 → 문화 변동 가능성

        ↓ 라운드 종료마다 반복

[Post-sim Review — 매 라운드 종료 시]
오케스트레이터 ↔ 작가: 문화 변동 여부 대화
  → 기존 문화 강화 / 약화 / 분기 / 소멸 합의
  → 합의된 내용으로 CULTURE 레코드 갱신
```

## 문화 영향 메커니즘

### 순응 스펙트럼

| 유형 | 비율(참고) | 행동 양상 |
|---|---|---|
| 순응자 | 다수 | 기존 문화를 자연스럽게 흡수, 재현 |
| 소극적 이탈자 | 일부 | 문화를 따르지 않으나 충돌은 회피 |
| 능동적 저항자 | 소수 | 문화 위반을 의도적으로 수행, 반응 유도 |
| 문화 파괴자 | 극소수 | 지속적 이탈로 문화 변동 가능성을 만들어냄 |

비율은 고정값이 아니며, 게시판 성격과 시드 이벤트에 따라 달라진다.

### 파괴자 감지 — 시스템 자동 추론

오케스트레이터가 P-* 빅5/파생지표 기반으로 문화 파괴자 후보를 자동 추론한다.
작가가 직접 태깅하지 않는다.

참고 기준 (확정값 아님, 시뮬 결과로 보정):
- `DT(어두운 3요소)` 상위 + `E(외향성)` 높음 → 어그로·정면 충돌 경향
- `O(개방성)` 높음 + `C(성실성)` 낮음 → 규범 경시·실험적 행동
- `A(우호성)` 낮음 + `N(신경증)` 높음 → 반응적 충돌 유발

### 작가 개입 조건

자연 창발이 원칙이나, 다음 조건에서 작가가 직접 개입할 수 있다:

1. **밋밋한 창발**: 시뮬이 여러 번 돌아도 게시판 간 성격 차이가 없을 때
   → 작가가 해당 게시판을 `crystallized` 상태로 직접 선언
2. **서사 필요**: 작품 전개상 어그로·갈등·분화가 필요할 때
   → "이 후보를 찾아서 이런 글을 작성하게 만들어라" 지시
   → 오케스트레이터가 조건에 맞는 P-* 후보 선별 후 실행

## 시뮬레이션 주입 방식

Layer B와의 핵심 차이: **"규칙"이 아니라 "경향"으로 서술한다.**

```
Layer B 주입 방식 (규칙):
  "이 게시판은 완전 익명이며, 반말이 기본이고,
   맥락 설명 없이 던지기가 문화다."

Layer A 주입 방식 (경향):
  "이 게시판에서는 반말이 지배적인 경향이 있다.
   대부분의 참여자가 자연스럽게 이를 따르지만,
   모든 참여자가 그러는 것은 아니다.
   존댓말을 쓰면 어색하게 여겨질 수 있으며,
   가끔 이를 지적하거나 담론으로 번지는 경우도 있다."
```

API 호출 시 위치: `system_prompt` 또는 `user_message`의 board context 항목에 포함.
강도(`strength`)에 따라 서술의 확신도를 조절한다.

| strength | 서술 톤 |
|---|---|
| weak | "~하는 경향이 조금 보인다" |
| moderate | "~가 지배적인 편이다" |
| strong | "~가 이 게시판의 주류 문화다. 이탈 시 반응을 유발할 수 있다." |

## CULTURE 레코드 포맷

```yaml
CULTURE-NNN:
  board_id: BOARD-XXX
  pattern_name: "한 줄 이름"
  observed_pattern: >
    지배적인 행동/어조/규범을 자연어로 기술.
    어떤 상황에서 발현되는지, 어떤 방식으로 재현되는지 포함.
  first_observed: "simrun-XXX"
  strength: weak / moderate / strong
  conformity_pressure: >
    문화 위반 시 발생하는 반응 기술.
    (예: 무시 / 가벼운 지적 / 담론화 / 집단 충돌)
  disruptor_profile: >
    이 문화에 저항하는 경향이 있는 빅5/파생지표 조건.
    오케스트레이터 추론 기반, 시뮬 결과로 보정.
  author_note: ""   # 작가 개입 시 메모. 없으면 비워둠.
  status: emergent  # emergent / crystallized
  evolution_log:
    - date: ""
      event: ""     # 문화 변동 사유 및 내용
```

### status 정의

| status | 설명 |
|---|---|
| `emergent` | 시뮬 결과 누적으로 자생 중. 변동 가능. |
| `crystallized` | 작가가 명시적으로 고정한 상태. 이후 시뮬에서 고정값으로 작동. |
