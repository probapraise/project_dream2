## 요약(선택)
- [PUBLIC] 각인광장은 한 번에 완성형으로 열리는 게시판이 아니라, `조회 전용 -> 보충 -> 질문 -> 검색 -> 추천/정렬 -> 관리/정화 -> 고밀도 기록 -> 현상 열람실 -> 영상 각인 플랫폼` 순으로 확장되는 인프라다.
- [PUBLIC] 기본 구조는 `키리온의 관리자 콘솔`, `성목 분지 공용 런타임`, `배지 표준 단말`의 삼층이며, `현상 열람실`은 고정식 예외 설비다.
- [PUBLIC] 운영 상태는 `S0 비공식 운용 -> S1 감리 허가 -> S2 시범 운영 -> S3 드리아드 협약`으로 승급한다.
- [PUBLIC] 학생이 직접 다루는 것은 공개 기록, 보충, 질문, 검색, 피드백 같은 구조화 패킷이고, `정신 서명/질의 서명/가중치`는 자동 채널 또는 관리자층에 남는다.
- [CONFIDENTIAL] 후행 패턴 학습은 운영 규칙의 전면 기능이 아니라, 뒷단 추천/정렬과 의도 색인을 설명하기 위한 작가용 약칭 `LMM`으로만 유지한다.

## 본문
### 17.1 단계별 해금 구조
- [PUBLIC] 각인광장의 운영 규칙은 처음부터 모두 열려 있지 않다.

| 단계 | 학생이 직접 다루는 것 | 기본 운영 상태 | 비고 |
| --- | --- | --- | --- |
| `0 개인 도구` | 없음 | `S0 비공식 운용` | 완전 개인 운용 |
| `1 조회 전용` | 공개 기록 열람 | `S1 감리 허가` | 감리 허가의 첫 구현 |
| `2 보충 허용` | 보충 패킷 | `S2 시범 운영` | 새 글 폭증 방지 |
| `3 새 질문 허용` | 질문 패킷 | `S2 시범 운영` | 본체가 물음에 반응 |
| `4 검색 확대` | 검색 패킷 | `S2 시범 운영` | 학생 자율 탐색 시작 |
| `5 추천/정렬` | 피드백 패킷 | `S3 드리아드 협약` | 자연 가중치 표면화 |
| `6 관리/정화` | 읽기 위주 | `S3 드리아드 협약` | 상태 표시와 격리 제도화 |
| `7 고밀도 기록` | richer view only | `S3 드리아드 협약` | 총량 감리 필요 |
| `8 현상 열람실` | 현상 열람 패킷 열람 | `S3 드리아드 협약` | 고정식 설비 전용 |
| `9 영상 각인 플랫폼` | 영상 패킷 열람/제한 업로드 | `S3 드리아드 협약` | 허가 업로더와 할당량 필요 |

### 17.2 운영 상태기계
- [PUBLIC] 기능 해금과 운영 상태는 같은 개념이 아니다.
- [PUBLIC] `S0 비공식 운용`
  - 키리온이 그리모어와 요정 동행을 기반으로 개인 기록고를 굴리고, 필요할 때만 소수에게 우회적으로 정보를 뿌리는 단계다.
- [PUBLIC] `S1 감리 허가`
  - 드리아드가 피해 통제를 위해 제한된 분지를 공식 감리 대상으로 내어 주는 단계다. 조회 전용, 허가된 소수 사용자, 단일 줄기가 기본이다.
- [PUBLIC] `S2 시범 운영`
  - 아직 협약은 아니지만 입력과 탐색 일부를 제한적으로 여는 반공용 단계다. `보충 -> 질문 -> 검색`이 여기서 열린다.
- [PUBLIC] `S3 드리아드 협약`
  - 드리아드가 키리온의 구조를 `기를 만한 인프라`로 인정하고, 장기 보존과 추천/정렬, 관리/정화, 고밀도 기록, 현상 열람실, 영상 플랫폼을 제도권 범위에서 허용하는 단계다.
- [CONFIDENTIAL] 승급 기준은 본체의 욕구, 드리아드의 감리 판단, 키리온의 유지 능력이 함께 맞물린다. 실패 시에는 `S3 -> S2 -> S1` 순으로 기능을 회수하는 쪽이 기본이다.

### 17.3 구조 계층과 핵심 원칙
- [PUBLIC] `그리모어`
  - 키리온의 관리자 콘솔이다. 비공개 실타래, 구조 편집, 공개 범위, 출시 결정을 다룬다.
- [PUBLIC] `성목 분지`
  - 공용 런타임이다. 공개 기록 저장, 중계, 단계별 입력 수용을 맡는다.
- [PUBLIC] `배지`
  - 표준 단말이다. 허용된 형식을 그대로 표시하고 구조화된 입력만 보낸다.
- [PUBLIC] `현상 열람실`
  - 학술원 기존 환영 설비를 전용한 고정식 예외 설비다. 고밀도 복합 감각 기록을 되풀리지만, 루트 권한은 없다.
- [PUBLIC] `영상 각인 플랫폼`
  - 배지에서 재생 가능한 시청각 기록층이다. 다만 업로드는 허가된 인원만 할 수 있다.
- [PUBLIC] 핵심 원칙:
  - `저장 형식 = 표시 형식`이다. 배지는 레이아웃을 재해석하지 않는다.
  - `배지는 끝까지 얇다`. 똑똑해지는 것은 단말이 아니라 런타임과 저장 구조다.
  - `정신 서명`, `질의 서명`, `가중치`는 초반부터 학생 표면에 직접 노출하지 않는다.
- [PUBLIC] 그리모어와 성목 분지의 직접 연결은 자유 텍스트 전체를 그대로 흘려보내는 방식이 아니라, 관리자 계층이 정리한 `인장형 입력`을 보내는 방식에 가깝다.
- [PUBLIC] 그래서 학생 표면에는 검색/보충/질문 패킷이 보이더라도, 런타임 깊은 층에서는 `질의 인장`과 `기록 인장`처럼 더 좁고 구조화된 입력이 움직인다고 이해하면 된다.

### 17.4 공용 패킷과 권한
- [PUBLIC] `공개 기록 패킷`
  - visible fields: `제목`, `본문`, `공개 표식`, `상태 표시`, `최종 갱신 시각`
  - system fields: `record_id`, `branch_scope`, `author_signature`, `retention_class`, `release_source`
  - first open stage: `1 조회 전용`
  - write permission: 초기에는 `키리온만`
- [PUBLIC] `보충 패킷`
  - visible fields: `target_record_id`, `보충 본문`, `보충 유형(정정/추가 사례/현장 후기)`
  - system fields: `supplement_id`, `author_signature`, `timestamp`, `source_group`
  - first open stage: `2 보충 허용`
- [PUBLIC] `질문 패킷`
  - visible fields: `제목`, `짧은 본문`, `공개 표식`
  - system fields: `question_id`, `author_signature`, `branch_scope`, `priority_bucket`
  - first open stage: `3 새 질문 허용`
- [PUBLIC] `검색 패킷`
  - visible fields: `표식 선택`, `공개 줄기 선택`, `범위 선택(기숙사/실습조/권역)`
  - system fields: `query_id`, `query_signature`, `branch_scope`, `runtime_query_weight`
  - first open stage: `4 검색 확대`
- [PUBLIC] `피드백 패킷`
  - visible fields: `도움됨`, `관련 없음`, `구식 정보`
  - system fields: `feedback_id`, `target_record_id`, `actor_signature`, `applied_weight_delta`
  - first open stage: `5 추천/정렬`
- [PUBLIC] `상태 표시 패킷`
  - visible fields: `해결됨`, `주의`, `구식`, `제한`, `격리` 같은 상태값과 변경 이유 요약
  - system fields: `marker_id`, `actor_role`, `review_due`, `quarantine_scope`
  - write permission: 초기엔 `키리온/허가된 관리자만`
- [PUBLIC] `현상 열람 패킷`
  - visible fields: `제목`, `요약 문구`, `접근 등급`, `필요 마나량`, `열람 가능 위치`
  - system fields: `capsule_id`, `curator_signature`, `replay_density`, `storage_weight`, `room_binding`, `time_limit`
  - first open stage: `8 현상 열람실`
  - note: 배지에는 제목, 요약, 접근 등급만 뜨고 실제 되풀림은 열람실에서만 일어난다.
- [PUBLIC] `영상 패킷`
  - visible fields: `제목`, `짧은 설명`, `재생 길이`, `공개 범위`, `업로더 표식`, `썸네일 표식`
  - system fields: `video_id`, `author_signature`, `quota_class`, `catalyst_signature`, `duration_class`, `replay_cost`, `moderation_flag`
  - first open stage: `9 영상 각인 플랫폼`
  - write permission: `키리온 + 허가된 영상 업로더`, 공개 범위 확정은 관리자 계층

### 17.4.1 인장형 입출력(운영 메커니즘)
- [PUBLIC] `질의 인장`
  - 키리온 또는 런타임이 검색 줄기, 표제, 공개 표식, 범위를 구조화해 만든 질의 키다.
  - 성목 분지는 자유 형식 문장 전체를 해석하기보다, 이 `질의 인장`에 대응하는 기록 다발을 끌어올리는 식으로 반응한다.
- [PUBLIC] `기록 인장`
  - 공개하기로 확정한 기록을 저장 가능한 묶음으로 정리한 인장형 입력이다.
  - 요정 동행 이후 초기 외부 저장은 이 `기록 인장`이 성목 수심에 전달되는 방식으로 이해하는 편이 자연스럽다.
- [CONFIDENTIAL] 따라서 `query_signature`, `author_signature`, `release_source` 같은 시스템 필드는 표면 패킷 아래에서 인장형 입출력을 추적하는 운영 흔적에 가깝다.

### 17.5 관리자 전용 확장 필드와 자동 채널
- [CONFIDENTIAL] 그리모어 전용 확장 필드
  - `비공개 실타래 참조`: 아직 공개하지 않은 기록 다발과 전투용 임시 연결
  - `맥락 가중치 힌트`: 질의에서 무엇이 핵심인지 루트 계층이 직접 부여하는 힌트
  - `연관도 표식`: 기록 사이의 강/약 연결과 우선순위 차등
  - `출시 규칙`: 어느 분지, 어느 사용자군에 언제 풀지 정하는 관리자 필드
  - `보정 노트`: 후행 패턴 학습이 이상 징후를 보일 때 원인 추적에 쓰는 메모
  - `프로토타입 문법`: 공용화 전 그리모어 안에서만 시험하는 새 질의/표시 구조
- [CONFIDENTIAL] 자동 부착 채널
  - `author_signature`: 공개 기록, 보충, 질문에 자동 부착되는 저작 확인 채널
  - `query_signature`: 인증된 검색/질문 이벤트에 자동 부착되는 질의 채널
  - `runtime_weight`: 추천/정렬 이후 런타임이 내부적으로 유지하는 가중치 값
- [CONFIDENTIAL] 따라서 `정신 서명 분석 결과`, `맥락 가중치`, `연관도`, `보정 노트`는 학생 표면에 직접 드러내지 않는다.

### 17.6 운영 주체와 권한(요약)
- [PUBLIC] 키리온/관리자 계층
  - 공개 범위 결정, 상태 변경, 상위 봉문, 정화 루틴 설계, 공개 본문 확정, 출시 결정을 맡는다.
- [PUBLIC] 일반 학생 사용자
  - 단계별 허용 범위 안에서 읽고, 보충하고, 질문하고, 검색하고, 피드백을 남긴다.
- [PUBLIC] 허가된 현상 열람자
  - 자격 명부와 시간 제한 아래 `현상 열람실`을 사용할 수 있다.
- [PUBLIC] 허가된 영상 업로더
  - 교육을 마친 뒤 할당량 안에서 `영상 각인술` 기록을 올릴 수 있다.
- [PUBLIC] 학생회 공보국/감찰/교수회
  - 작성자 실체 확인, 봉문, 정지, 조사 목적 로그 열람, 키워드 제한 같은 제도권 개입을 담당한다.
- [PUBLIC] 엘프 클랜/성목 수호 측
  - 직접 게시판 운영을 맡지는 않더라도, 분지 건강과 중계 안정, 수심 접근, 승급 심사를 사실상 좌우한다.
- [CONFIDENTIAL] 따라서 실제 권력은 `누가 글을 쓰느냐`보다 `누가 수심/분지/로그/열람실/승급 판정권을 쥐느냐`에 더 자주 걸린다.

### 17.7 신원·표시명·익명성
- [PUBLIC] 각인광장의 기본 입력은 배지 연동 신원을 전제로 한다.
- [PUBLIC] 다만 게시판/권역에 따라 `표시명`이나 `별칭`을 허용할 수 있다. 이것은 플랫폼 전체의 기본 원리가 아니라 후행 운영 정책이다.
- [PUBLIC] 익명성이 강할수록 비용과 신고 민감도가 높아지는 방향은 자연스럽지만, 이는 모든 줄기에 일괄 적용되는 철칙이라기보다 운영자가 조절하는 보드 규칙에 가깝다.
- [CONFIDENTIAL] `가면 계정` 같은 고위험 익명 모델은 초기 광장 기본 규칙보다 후행 사건용·특수 보드용 장치로 두는 편이 정합적이다.

### 17.8 위반 처리·제재·항소 프로토콜
- [PUBLIC] 신고/자동탐지 -> 임시 조치(가리기/봉문/격리) -> 증거 요청(원문/정본/로그) -> 판정(L1~L5) -> 항소.
- [PUBLIC] 항소는 정본/준정본 증거가 강할수록 유리하지만, 그 증거를 만드는 데도 열람권과 비용이 든다.
- [PUBLIC] 영상 각인 위반은 일반 글 위반보다 무겁게 다루는 편이 자연스럽다. 촉매 규격, 길이 제한, 업로드 할당량, 열람 범위가 모두 로그로 남기 때문이다.
- [PUBLIC] 현상 열람실 위반은 게시판 징계만이 아니라 시설 접근 제한으로 바로 이어질 수 있다.

### 17.9 조작/선동 기법(사건용)
- 보존권 폭격
  - 특정 글을 보존권으로 도배해 노출 탭을 장악.
- 발췌각인 짜깁기
  - 원문 일부만 공유해 프레임을 고정.
- 봉문 유도
  - 금기 키워드나 서명 단서로 봉문을 유도해 음모론 연료로 사용.
- 영상 프레이밍
