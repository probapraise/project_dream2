# logic_review_rubric_v1

- scope: `cycle_02 raw draft review`
- role: `draft-time discourse and logic check`

## Review categories

- `contrast`
  - 실제로 충돌하지 않는 사실을 `하지만`, `말과 달리`, `아니었다` 식으로 억지 대비하지 않았는가
- `redundancy`
  - 이미 보여 준 인상을 같은 문장군 안에서 다시 요약하지 않았는가
- `inference`
  - 관찰 하나에서 세계 총평, 계급 총평, 인물 본심까지 너무 빨리 점프하지 않았는가
- `dialogue`
  - 뜻은 맞지만 입말로는 어색한 기능 대사가 끼지 않았는가
- `pov plausibility`
  - 키리온이 그 자리에서 실제로 그렇게까지 정리할 수 있는지, 납득이 약하지 않은가
- `scene pressure`
  - 정보는 이해되지만 몸 압박이 빠져 `정리된 해설`처럼 읽히지 않는가

## Severity guide

- `major`
  - 사용자 초고로 내보내기 전에 반드시 고쳐야 하는 문제
- `moderate`
  - 구조는 유지해도 되지만, 다음 패스에서 고치는 편이 확실히 나은 문제
- `minor`
  - 사용자 quick edit에 넘겨도 되지만, 가능하면 내부에서 먼저 정리할 수 있는 문제

## Output rule

- review 문서는 `findings first`로 쓴다.
- 문장을 바로 고치기보다 `무슨 종류의 문제가 어디에 있는지`를 먼저 명시한다.
- rewrite 단계는 review 이후에만 수행한다.
