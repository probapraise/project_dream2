# Stage 1 Notes: 서고색인과 색인의 그리모어

- change_id: CR-20260315-002
- status: draft
- focus: first miracle / stage 1 artifact

## 1. 확정 방향
- `백지의 그리모어`는 처음부터 완성형 만능 마도서가 아니다.
- 1차 원형은 `서고색인` 마법을 담은 고급 상용 아티팩트다.
- 작중 표면에서 이 물건은 `색인의 그리모어` 계열로 불린다.
- `백지의 그리모어`는 키리온이 이후 자기 식으로 성장/개조시킨 뒤 붙는 별칭 후보로 남긴다.

## 2. 서고의 세계관 정의
- 이 세계의 `서고`와 `도서관`은 단순한 장서 대여 장소가 아니다.
- 지역 내에서 생산된 핵심 문서, 등록 문서, 정본, 봉인 기록, 제한 열람 문서를 보관하는 `문서 금고` 역할을 수행한다.
- 따라서 서고는 지식기관이면서 동시에 권한 기관이고, 열람권은 자격/신분/직무/서약에 따라 엄격히 제한된다.
- 문서 기반 사회이므로, 서고 운영과 문서 위치 관리 자체가 행정력과 권력의 일부다.

## 3. 왜 아티팩트가 필요한가
- `서고색인` 자체는 원래 마법사가 시전할 수 있는 등록식이다.
- 하지만 극비 문서의 정확한 위치, 존재 여부, 열람권 체계는 그 자체로 민감 정보다.
- 이 정보를 전담 마법사에게 상시 노출하는 것은 배신, 매수, 강요, 기억 추적의 위험을 만든다.
- 그래서 대형 서고는 `배신할 걱정이 없는 기능 분리형 단말`로서 색인 아티팩트를 배치한다.
- 아티팩트는 필요한 색인 기능만 수행하고, 권한 판정은 별도 인프라 레이어와 연동한다.

## 4. 색인의 그리모어의 존재 이유
- 장서/문서 위치 탐색을 표준화한다.
- 배지, 마나 사인, 열람권 레지스트리와 연동해 사용자별 반환 범위를 제한한다.
- 극비 문서 존재 여부를 사람 대신 단말이 판정하게 해 내부 누설 위험을 줄인다.
- 숙련 마법사를 상시 대기시키는 것보다 반복 업무와 보안 측면에서 유리하다.

## 4-A. 엄밀한 기능 분리
- `서고색인`과 `열람권 판정`은 엄밀히 말해 같은 기능이 아니다.
- `서고색인`
  - 문서의 표제, 등록번호, 발췌문, 출처 사슬을 바탕으로 `어디에 있는가`를 추적하는 등록식
  - 본질은 위치/연결 고정이다.
- `열람권 판정`
  - 사용자의 등록된 마나 사인과 서고측 인가 목록을 대조해 `그 정보를 보여줘도 되는가`를 판정하는 별도 인프라 기능
  - 본질은 접근 제어다.
- 따라서 `색인의 그리모어`는 한 개의 마법을 담은 순수 도구라기보다,
  - 핵심 마법: `서고색인`
  - 연동 레이어: `열람권 판정`
  - 사용자 식별 레이어: 등록된 마나 사인
  를 묶어 상품화한 고급 서고 단말로 보는 편이 정합적이다.

## 4-B. 이 분리가 주는 장점
- 색인 기능 자체는 비교적 널리 보급할 수 있고, 권한 제어는 기관별로 별도 연결할 수 있다.
- 키리온이 나중에 탐내는 것도 `권한 시스템 전체`가 아니라 `색인 엔진의 핵심부`로 좁혀져 더 자연스럽다.
- 이후 주인공이 이를 개조해 `문서 위치 색인 -> 발언/기록 색인 -> 광장 색인`으로 확장하는 발판이 된다.
- 학술원 전용 인프라인 `배지`와 왕도/대도시 서고 인프라를 굳이 묶지 않아도 되므로 세계관 계층이 더 깨끗해진다.

## 4-C. 현행 판정 규칙 확정
- 왕도와 대도시 서고에 배치된 `색인의 그리모어`는 `배지`를 사용하지 않는다.
- 사용자가 손을 대거나 시동 문구를 입력하면, 그리모어는 사용자의 `마나 사인`을 읽는다.
- 그 마나 사인이 서고측 등록 목록과 일치할 때만 허용 범위 안의 위치를 반환한다.
- 등록되지 않은 마나 사인 또는 허가되지 않은 접근은 `열람 불가` 판정으로 막는다.
- 즉 표면적으로는 `서고색인` 하나처럼 보이지만, 실제 운용상으로는 `색인 + 사인 대조 + 반환 차단`이 결합된 등록식 아티팩트다.

## 5. 현재 추천 정의
- 정식 마법명: `서고색인`
- 분류: 등록식
- 상용 아티팩트명: `색인의 그리모어`
- 보급 레벨:
  - 왕도 왕립도서관
  - 대도시급 공공 도서관
  - 대형 학술기관
  - 대귀족가의 핵심 서고
- 제작 성격:
  - 값비싼 수제 마도구이며, 현대적 의미의 `개인용 / 산업용 / 인프라용` 표준 모델 라인업은 없다.
  - 각 `색인의 그리모어`는 연동될 서고, 장서대장, 인가 규칙에 맞춰 조정 제작된다.
  - 따라서 차이는 SKU나 스펙 등급보다 `어느 레지스트리에 물려 있느냐`, `어떤 인가 규칙이 새겨졌느냐`에서 발생한다.

## 6. 초반 기능 제한
- 현재 위치 검색만 수행한다.
- 입력값은 표제, 등록 번호, 저자명, 발췌문 같은 제한된 항목으로 잡는다.
- 초반에는 문서 평가, 의미 해석, 요약, 추천 같은 기능이 없다.
- 즉 `검색 엔진`이 아니라 `마나 사인 등록 기반 서고 위치 단말`이다.

## 7. 주인공이 탐낼 근거
- 남들에게는 비싼 편의 장비지만, 키리온에게는 `입력 -> 색인 -> 반환 -> 권한 제어` 구조를 가진 정보 인프라 씨앗으로 보인다.
- 현재 키리온은 공개 마법 수련보다 정보와 기록 쪽에서 합법적으로 힘을 축적해야 한다.
- 따라서 무기보다 이런 색인 아티팩트에 먼저 군침을 흘리는 편이 캐릭터와 세계관 모두에 맞다.

## 8. 후속 설계 쟁점
- `서고색인`의 입력값/출력값 구체화
- 왕도에서 판매 가능한 범위가 `완성품`인지, 아니면 특정 서고에 연동하기 전 단계의 `미인장 본체`인지
- 키리온이 왕도에서 이 물건을 보고 발상 전환을 일으키는 장면 설계
- 이후 `색인의 그리모어 -> 백지의 그리모어`로 진화하는 계기 설계
