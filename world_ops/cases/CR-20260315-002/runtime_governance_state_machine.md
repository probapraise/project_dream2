# Runtime Governance State Machine

- change_id: CR-20260315-002
- status: draft
- focus: `비공식 운용 -> 감리 허가 -> 시범 운영 -> 드리아드 협약`의 거버넌스 상태를 고정한다

## 1. 문서 목적
- `드리아드 감리 허가`와 `드리아드 협약` 사이에 실제 운영 상태가 하나 더 있다는 점을 명시한다.
- 기능 해금 단계와 운영 상태를 분리해, `무엇이 가능하냐`와 `어떤 통치 상태냐`를 혼동하지 않게 한다.
- live SSOT 재작성 시 `공용화가 곧바로 완성형 플랫폼`처럼 읽히는 문제를 막는다.

## 2. 상태 개요

| 상태 | 성격 | 커뮤니티 단계 매핑 | 운영 키워드 |
|---|---|---|---|
| S0 비공식 운용 | 개인 기연의 그림자 공유 | 단계 0 | 우회, 즉흥, 비표준 |
| S1 감리 허가 | 제한된 공식 줄기 | 단계 1 | 조회 전용, 단일 줄기, 즉시 차단 가능 |
| S2 시범 운영 | 제한적 반공용 런타임 | 단계 2~4 | 보충/질문/검색, 감리 지속, 승급 심사 |
| S3 드리아드 협약 | 제도화된 로컬 인프라 | 단계 5~9 | 장기 보존, 추천/정렬, 관리 체계, 현상 열람, 영상 업로드 |

## 3. 상태 정의

### 3.1 S0 `비공식 운용`
- 정의:
  - 키리온이 그리모어와 요정 동행을 기반으로 개인 기록고를 굴리고,
    필요할 때만 소수에게 우회적으로 정보를 뿌리는 단계
- 허용:
  - 키리온 단독 검색
  - 키리온 단독 저장
  - 비공식 전술 전달
  - 요정 거래와 외부 저장고 사용
- 금지:
  - 공식 분지 배정
  - 학생 공개 입력
  - 장기 보존권 주장
- 드리아드의 입장:
  - 감시 대상
  - 잠재적 오염원
- S1 진입 조건:
  - 성목 본체가 반응해 완전 차단이 불가능해짐
  - 비공식 유입이 이미 발생함
  - 키리온이 흐름을 `한 줄기`로 묶어 감리 가능한 형태를 제시함

### 3.2 S1 `감리 허가`
- 정의:
  - 드리아드가 피해 통제를 위해 제한된 분지를 공식 감리 대상으로 내어 주는 단계
- 허용:
  - `조회 전용`
  - 허가된 소수 사용자의 배지 열람
  - 단일 줄기 또는 극소수 분지 사용
  - 기록 공개 범위의 관리자 결정
- 금지:
  - 학생 직접 입력
  - 학생 독자 검색
  - 학생 분류 개편
  - 다중 분지 확장
- 드리아드의 통제권:
  - 언제든 줄기를 닫을 수 있음
  - 어떤 기록이 흐르는지 실시간 감리
  - 사용자군을 좁게 제한
- 승급 조건(S2):
  - 줄기 건강이 안정적임
  - 전술 열람의 실효가 검증됨
  - 키리온이 기록 정리와 공개 범위 관리를 지속적으로 수행함
  - 본체가 `다른 목소리`와 `새 물음`을 더 원하기 시작함
- 강등/회수 조건:
  - 무단 입력 시도 누적
  - 분지 오염, 과부하, 누수
  - 키리온이 유지 책임을 방치함

### 3.3 S2 `시범 운영`
- 정의:
  - 아직 협약은 아니지만, 입력과 탐색 일부를 제한적으로 여는 반공용 운영 단계
- 허용:
  - `보충 허용`
  - `새 질문 허용`
  - `검색 자유도 확대`
  - 실험적 공용 문법 적용
  - 한정된 사용자군 확장
- 금지:
  - 무제한 익명 난입
  - 전면 공개 광장화
  - 자유로운 감각 기록 업로드
  - 장기 슬롯 무제한 사용
- 드리아드의 입장:
  - 여전히 관리자가 우선
  - 하지만 `막아야 할 흐름`과 `길러도 되는 흐름`을 구분하기 시작함
- 키리온의 의무:
  - 부작용 수습
  - 묵은 기록 정리
  - 메타 색인 유지
  - 질문/보충 구조 정돈
- 승급 조건(S3):
  - 공용 문법이 안정적으로 굴러감
  - 기록 정리/정화 루틴이 검증됨
  - 검색 확장이 성목 부담을 줄이는 방향으로 작동함
  - 단일 실습조를 넘어 로컬 커뮤니티로 확장할 가치가 증명됨
- 강등 조건:
  - 허위/구식 기록이 실전 피해를 일으킴
  - 가지 정리와 격리 루틴이 작동하지 않음
  - 질문/검색 폭증으로 분지 건강이 악화됨

### 3.4 S3 `드리아드 협약`
- 정의:
  - 드리아드가 키리온의 구조를 `기를 만한 인프라`로 인정하고 제도적 사용권을 부여한 상태
- 허용:
  - 장기 보존 슬롯
  - 로컬 커뮤니티 단위의 안정적 운용
  - 추천/정렬
  - 관리/정화
  - 고밀도 기록 운용
  - 고정식 현상 열람실 운용
  - 제한된 다중 업로더 영상 각인 운용
  - 제한적 역할 위임
- 여전히 금지:
  - 무제한 혼돈형 공개 광장화
  - 그리모어 루트 접근 공유
  - 감각 기록 무제한 업로드
  - 영상 각인 무제한 업로드
  - 개인 휴대형 만능 감각 단말 보급
  - 드리아드/성목 승인 없는 타 권역 침범
- 드리아드의 입장:
  - 감리자에서 공동 운영 파트너로 이동
  - 나무를 지키는 권한은 계속 유지
- 추가 운영 조건:
  - 현상 열람은 학술원 기존 환영 설비를 전용한 고정식 열람실에서만 허용
  - 열람 자격 명부와 시간 제한이 필요함
  - 열람자가 내는 마나는 재생 비용으로만 쓰이며 저장 총량 문제를 면제하지 않음
  - 영상 각인 업로더는 전공/허가/할당량 기준으로 관리됨
  - 영상 각인 촉매 규격과 길이 제한을 어기면 즉시 회수 가능
- 이후 분기:
  - 성목 본체 공명 심화
  - LMM 고도화
  - 엘프 클랜 및 전심학파와의 정치적 긴장 심화

## 4. 기능 해금과의 매핑

| 기능 단계 | 기본 운영 상태 | 비고 |
|---|---|---|
| 단계 0 개인 도구 | S0 | 완전 개인 운용 |
| 단계 1 조회 전용 | S1 | 감리 허가의 첫 구현 |
| 단계 2 보충 허용 | S2 | 시범 운영 진입 |
| 단계 3 새 질문 허용 | S2 | 본체의 물음 선호가 반영됨 |
| 단계 4 검색 확대 | S2 | 기술적 돌파가 승급 심사 재료가 됨 |
| 단계 5 추천/정렬 | S3 | 제도권 인프라화의 시작 |
| 단계 6 관리/정화 | S3 | 운영 책임이 제도화됨 |
| 단계 7 고밀도 기록 | S3 | 총량 감리와 권역 사용권이 필요함 |
| 단계 8 현상 열람실 | S3 | 기존 환영 설비 전용, 자격 명부, 열람 마나 규정이 필요함 |
| 단계 9 영상 각인 플랫폼 | S3 | 업로더 할당량, 촉매 규격, 길이 제한, 로그 추적이 필요함 |

## 5. 전환 조건의 성격
- `S0 -> S1`
  - 본체의 욕구가 드리아드의 손을 묶는 전환
- `S1 -> S2`
  - 본체의 욕구 + 키리온의 유지 능력이 결합하는 전환
- `S2 -> S3`
  - 순수 호기심 단계가 끝나고, 구조와 운영이 제도권 인프라로 인정받는 전환

## 6. 책임 분배

### 6.1 키리온
- 공용 문법 설계
- 공개 범위 결정
- 부작용 수습
- 기록 정리와 검색 구조 유지

### 6.2 드리아드
- 분지 건강 감리
- 슬롯 배정
- 승급/강등 판정
- 과도한 흐름 차단

### 6.3 성목 본체
- 근본 동력 제공
- 완전 차단 불가의 압력 형성
- 후반 공명/LMM 심화의 상위 배경

### 6.4 학생 사용자
- 단계별 허용 문법 사용
- 보충/질문/피드백 제공
- 허가 시 현상 열람실 사용
- 허가 시 영상 각인 업로드
- 잘못된 정보와 과잉 흐름의 실전 리스크를 함께 짊어짐

## 7. 롤백 원칙
- `S3 -> S2`
  - 장기 보존과 추천/정렬은 유지하되, 영상 업로드와 현상 열람실부터 먼저 잠근다
- `S2 -> S1`
  - 직접 입력을 닫고 조회 전용으로 후퇴시킨다
- `S1 -> S0`
  - 공식 줄기를 닫고 다시 비공식 운용만 남긴다
- note:
  - 본체가 계속 관심을 보이는 한, 드리아드는 `완전 삭제`보다 `축소와 격리`를 우선한다

## 8. 현재 추천 결론
- `감리 허가`와 `협약`을 같은 상태로 취급하면 중간 성장의 손맛이 사라진다.
- `시범 운영`을 별도 상태로 두면 `보충 -> 질문 -> 검색 확대`가 모두 의미 있는 승급 시험이 된다.
- 이 상태 기계를 고정해 두면, 이후 live 문서에서 공용화와 정치적 파장을 훨씬 일관되게 서술할 수 있다.
