# Runtime Interface Contract

- change_id: CR-20260315-002
- status: draft
- focus: `그리모어 / 성목 분지 / 배지 / 현상 열람실` 사이의 입력 계층과 필드 권한을 고정한다

## 1. 문서 목적
- 후속 문서가 `공용 표면 문법`과 `그리모어 전용 확장 문법`을 섞어 쓰지 않게 만든다.
- `조회 전용 -> 보충 -> 질문 -> 검색 -> 추천/정렬 -> 관리/정화 -> 고밀도 기록 -> 현상 열람실 -> 영상 각인 플랫폼` 해금 순서와 LMM 시드 설계를 하나의 인터페이스 계약으로 묶는다.
- `정신 서명`, `연관도 표식`, `맥락 가중치`, `피드백 루프`가 어느 계층에 속하는지 명시한다.

## 2. 구조 삼층 + 예외 설비
- `그리모어`
  - 키리온만 쓰는 루트 콘솔
  - 자유 형식 실타래 질의, 비공개 기록, 구조 편집, 출시 결정 담당
- `성목 분지`
  - 공용 런타임
  - 공개 기록 저장, 중계, 권한 단계별 입력 수용 담당
- `배지`
  - 사용자 단말
  - 저장된 형식을 그대로 투사하고, 허용된 구조화 입력만 보낸다
- `현상 열람실`
  - 학술원 기존 고감각 환영 설비를 전용한 고정식 열람 시설
  - 성목 분지의 공용 런타임에 붙지만, 배지보다 높은 감각 밀도의 출력만 담당한다
  - 루트 접근권은 없고, 허가된 `현상 열람 패킷`만 재생한다

## 3. 핵심 원칙
- `저장 형식 = 표시 형식`
  - 배지는 레이아웃을 재해석하지 않는다.
  - 키리온이 무엇을 어떤 형태로 저장했는지가 그대로 사용감이 된다.
- `정신 서명 = 자동 부착 채널`
  - 사용자가 직접 보거나 고르거나 조작하는 항목이 아니다.
  - 인증된 질의/기록 이벤트에 시스템이 자동으로 붙인다.
- `맥락 가중치 / 연관도 / 보정 필드 = 기본적으로 관리자층`
  - 학생 표면에 그대로 노출하지 않는다.
  - 초반에는 그리모어만 직접 다루고, 후반에는 런타임이 내부적으로 소비한다.
- `배지는 끝까지 얇다`
  - 배지가 똑똑해지는 게 아니라 런타임과 저장 구조가 정교해진다.
- `현상 열람실도 사용자층이다`
  - 열람실은 출력 밀도를 높일 뿐, 관리자 권한이나 루트 질의를 주지 않는다.
- `열람 마나는 재생 비용이다`
  - 학생이 내는 마나는 고밀도 기록을 다시 풀어 올리는 데 쓰인다.
  - 영구 저장 부담과 슬롯 총량은 계속 드리아드 감리 대상이다.
- `영상 각인술은 제도 등록식이다`
  - 그리모어의 등록식 변질과 달리, 영상 각인술은 교수 검증과 교육 절차를 거친 의도적 등록식이다.
  - 따라서 제한된 다중 업로더 구조를 가질 수 있다.

## 4. 표면 패킷과 권한

### 4.1 `공개 기록 패킷`
- purpose:
  - 조회 전용 단계에서 학생이 읽는 기본 단위
- visible_fields:
  - 제목
  - 본문
  - 공개 표식
  - 상태 표시
  - 최종 갱신 시각
- system_fields:
  - record_id
  - branch_scope
  - author_signature
  - retention_class
  - release_source
- write_permission:
  - 초기에는 `키리온만`
  - 이후에도 공개 본문 확정과 출시 결정은 관리자 계층

### 4.2 `보충 패킷`
- purpose:
  - 기존 공개 기록 아래에 정정, 추가 사례, 현장 후기를 누적한다
- visible_fields:
  - target_record_id
  - 보충 본문
  - 보충 유형(`정정`, `추가 사례`, `현장 후기`)
- system_fields:
  - supplement_id
  - author_signature
  - timestamp
  - source_group
- first_open_stage:
  - 단계 2 `보충 허용`

### 4.3 `질문 패킷`
- purpose:
  - 아직 답이 없거나 기존 기록으로 회수되지 않는 새 물음을 올린다
- visible_fields:
  - 제목
  - 짧은 본문
  - 공개 표식
- system_fields:
  - question_id
  - author_signature
  - branch_scope
  - priority_bucket
- first_open_stage:
  - 단계 3 `새 질문 허용`

### 4.4 `검색 패킷`
- purpose:
  - 공개 실타래와 표식을 따라 기존 기록을 탐색한다
- visible_fields:
  - 표식 선택
  - 공개 줄기 선택
  - 범위 선택(`기숙사`, `실습조`, `권역`)
- system_fields:
  - query_id
  - query_signature
  - branch_scope
  - runtime_query_weight
- first_open_stage:
  - 단계 4 `검색 자유도 확대`

### 4.5 `피드백 패킷`
- purpose:
  - 추천/정렬과 후반 LMM 고도화를 위한 간단한 반응값을 수집한다
- visible_fields:
  - `도움됨`
  - `관련 없음`
  - `구식 정보`
- system_fields:
  - feedback_id
  - target_record_id
  - actor_signature
  - applied_weight_delta
- first_open_stage:
  - 단계 5 `추천/정렬`
- note:
  - 표면은 단순하지만, 내부적으로는 피드백 루프의 재료가 된다.

### 4.6 `상태 표시 패킷`
- purpose:
  - 기록을 `해결됨`, `주의`, `구식`, `제한`, `격리` 등으로 표시한다
- visible_fields:
  - 상태값
  - 상태 변경 이유 요약
- system_fields:
  - marker_id
  - actor_role
  - review_due
  - quarantine_scope
- write_permission:
  - 초기엔 `키리온/허가된 관리자만`
  - 학생은 읽기만 허용

### 4.7 `현상 열람 패킷`
- purpose:
  - 복합 감각 기록을 `현상 열람실`에서만 재생 가능한 형태로 묶는다
- visible_fields:
  - 제목
  - 요약 문구
  - 접근 등급(`공개 하이라이트`, `제한 공개`, `봉인`)
  - 필요 마나량
  - 열람 가능 위치
- system_fields:
  - capsule_id
  - curator_signature
  - replay_density
  - storage_weight
  - room_binding
  - time_limit
- write_permission:
  - 업로드/출시 결정은 `키리온만`
  - 열람은 허가된 자격 명부 + 고정식 열람실에서만
- first_open_stage:
  - 단계 8 `현상 열람실`
- note:
  - 배지에는 제목, 요약, 접근 등급만 뜨고 실제 되풀림은 현상 열람실에서만 일어난다.

### 4.8 `영상 패킷`
- purpose:
  - `영상 각인술`로 남긴 시각/청각 기록을 배지에서도 재생 가능한 형태로 묶는다
- visible_fields:
  - 제목
  - 짧은 설명
  - 재생 길이
  - 공개 범위
  - 업로더 표식
  - 썸네일 표식
- system_fields:
  - video_id
  - author_signature
  - quota_class
  - catalyst_signature
  - duration_class
  - replay_cost
  - moderation_flag
- write_permission:
  - 기본 업로드는 `키리온 + 허가된 영상 업로더`
  - 공개 범위 확정과 상위 봉인은 관리자 계층
- first_open_stage:
  - 단계 9 `영상 각인 플랫폼`
- note:
  - 배지에서도 재생 가능하지만, 학생별 업로드 할당량과 촉매 규격 제한이 붙는다.

## 5. 그리모어 전용 확장 필드
- `비공개 실타래 참조`
  - 아직 공개하지 않은 기록 다발, 보정 노트, 전투용 임시 연결
- `맥락 가중치 힌트`
  - 질의에서 무엇이 핵심인지 루트 계층이 직접 부여하는 힌트
- `연관도 표식`
  - 기록 A와 B의 강/약 연결, 유사 사례 묶음, 우선순위 차등
- `출시 규칙`
  - 어떤 기록을 어느 분지, 어느 사용자군에 언제 풀지 정하는 필드
- `보정 노트`
  - LMM이 후반부에 이상 패턴을 보일 때, 키리온이 원인 추적에 쓰는 관리자 메모
- `프로토타입 문법`
  - 공용화 전 그리모어 안에서만 시험하는 새 질의/표시 구조

## 6. 자동 부착 채널

### 6.1 `author_signature`
- 공개 기록, 보충, 질문에 자동 부착된다.
- 위조 불가 저작 확인과 후반 서명 기반 군집화의 재료다.
- 학생 표면에는 직접 노출하지 않는다.

### 6.2 `query_signature`
- 인증된 검색/질문 이벤트에 자동 부착된다.
- 후반부 `의도 색인`의 입력 채널로 쓰인다.
- 사용자가 선택하거나 조절할 수 없다.

### 6.3 `runtime_weight`
- 추천/정렬 단계 이후 런타임이 내부적으로 유지하는 가중치 값이다.
- 표면에서 직접 수정하지 않는다.
- `도움됨 / 관련 없음` 피드백, 반복 조회, 연속 보충이 누적되며 바뀐다.

## 7. 단계별 개방 매핑

| 단계 | 학생이 직접 다루는 패킷 | 관리자/그리모어만 다루는 것 | 비고 |
|---|---|---|---|
| 0 개인 도구 | 없음 | 공개 기록 작성, 비공개 기록, 실타래 질의, 출시 결정 | 완전 개인 운용 |
| 1 조회 전용 | 공개 기록 열람 | 검색 결정, 공개 범위 지정, 상태 표시 | 전술 열람망 |
| 2 보충 허용 | 보충 패킷 | 보충 승인/정리, 상태 표시 | 새 글 폭증 방지 |
| 3 새 질문 허용 | 질문 패킷 | 질문 분류, 공개 여부 결정 | 본체가 물음에 반응 |
| 4 검색 확대 | 검색 패킷 | 메타 색인 구조, 연관도 표식 | 학생 자율 탐색 시작 |
| 5 추천/정렬 | 피드백 패킷 | 추천 로직 보정, 정렬 규칙 | 자연 가중치 표면화 |
| 6 관리/정화 | 읽기 위주 | 상태 표시, 격리, 봉인, 정화 | 부작용 대응 |
| 7 고밀도 기록 | richer view only | 고밀도 저장 구조, 총량 감리 | 지도/약도/정지 이미지 확장 |
| 8 현상 열람실 | 현상 열람 패킷 열람 | 등급 지정, 자격 명부, 총량 감리, 열람실 운영 | 복합 감각 기록 재생 |
| 9 영상 각인 플랫폼 | 영상 패킷 열람/제한 업로드 | 업로드권 배분, 촉매 규격, 할당량, 정화 | 다중 업로더 시청각 기록 |

## 8. LMM 시드와 직접 연결되는 부분
- `맥락 가중치 힌트`
  - 초반에는 그리모어 전용, 후반에는 런타임 내부 필드로 확장
- `연관도 표식`
  - 그리모어가 먼저 설계하고, 이후 성목이 학습 재료로 활용
- `피드백 패킷`
  - 학생 표면에서는 단순한 반응값이지만, 내부적으로는 강화/약화 경로가 된다
- `author_signature`, `query_signature`
  - 사용자는 의식하지 않지만, 후반 멀티모달 LMM의 핵심 입력 채널
- `현상 열람 패킷`
  - 희소하지만 밀도 높은 복합 감각 기록은 후반 LMM의 기준선 데이터를 제공한다
- `영상 패킷`
  - 대량의 시청각 기록과 피드백 로그는 후반 LMM의 주된 스케일 데이터를 제공한다

## 9. 금지선
- 학생 표면에 `정신 서명 분석 결과`를 직접 띄우지 않는다.
- 학생에게 `맥락 가중치`, `연관도`, `보정 노트` 편집권을 주지 않는다.
- 배지 쪽에 즉석 UI 재배치나 독자 추론 로직을 두지 않는다.
- 현상 열람실에 루트 질의, 자유 업로드, 개인 휴대형 복제를 허용하지 않는다.
- 영상 각인 플랫폼을 무제한 업로드 광장처럼 열지 않는다.
- `의도 색인`을 감정 읽기, 거짓말 탐지, 독심술처럼 오해하게 만드는 문구를 금지한다.

## 10. 현재 추천 결론
- 공용 문법은 끝까지 `얇은 표면`으로 유지하는 편이 맞다.
- 현상 열람실은 `배지의 예외적 확장`이지, 별도 루트 콘솔이 아니다.
- 영상 각인 플랫폼은 그 다음 단계의 `제한적 다중 업로더 레인`으로 두는 편이 맞다.
- 깊은 의미 해석과 구조 보정은 `그리모어 루트 -> 성목 런타임 내부`에서 처리하고, 배지와 현상 열람실, 영상 플랫폼은 그 결과만 받는다.
- 이렇게 해야 `초반 전술 열람망`, `중반 질문/검색`, `후반 추천/LMM`, `현상 열람`, `영상 플랫폼`이 같은 계열 문법 위에서 자연스럽게 이어진다.
