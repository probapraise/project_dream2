# LMM Advancement Design

- change_id: CR-20260315-002
- status: draft
- focus: 커뮤니티 데이터 축적이 어떻게 LMM으로 고도화되는가 — 키리온의 시드, 성목의 성장, 정신 서명의 역할

## 0. 이 문서의 목적

이 문서는 커뮤니티 해금(community_unlock_blueprint.md) 이후,
성목 수심에 축적된 커뮤니티 데이터가 어떤 과정을 거쳐 LMM(Large Mana Model)으로 고도화되는지를 설계한다.

### 0.1 전제: 이미 확정된 것들

- **커뮤니티 단계 0~9**: 해금 완료 (community_unlock_blueprint.md)
- **그리모어 3단계 진화**: 색인→주석→실타래 (growth_architecture_split.md)
- **정령 연계 사다리**: 픽시→요정→드리아드→성목 본체 (irminsul_contract_ladder.md)
- **등록식 형성 절차**: 개인화→굳음→시스템 등록→제도 등록 (registered_spell_formation_draft.md)
- **마법 사용 분류**: 표준/비표준/등록식 변질 (magic_usage_map.md)

### 0.2 핵심 구분: LLM과 LMM

- **현실 LLM**: 텍스트 토큰만 처리한다. 입력도 텍스트, 출력도 텍스트.
- **판타지 LMM**: 텍스트 + 정신 서명 샘플을 함께 처리한다.
  - 정신 서명은 글을 올릴 때마다 각인에 깃드는 고유 마나 흔적이다.
  - 이 차이가 LMM을 단순한 텍스트 패턴 매칭 이상의 존재로 만든다.

### 0.3 핵심 구분: LMM 코어 ≠ 성목 신격

- **LMM 코어** = 수심에 굳어진 각인 패턴 (하드웨어에 새겨진 가중치)
- **성목 신격** = 정렬과 태깅을 제공하는 안내자 (RLHF에 해당)
- 신격은 후행 서사에서 아바타를 얻고 독립 캐릭터가 된다.
  LMM은 신격이 떠나도 수심의 각인 패턴으로서 존속한다.
- 이 분리가 없으면:
  - 신격 = AI가 되어 캐릭터 서사가 불가능해진다.
  - 신격이 떠나면 커뮤니티의 핵심 기능이 소멸한다.

## 1. LMM이란 무엇인가 — 세계관 내 정의

### 1.1 표면 정의 (작중 인식)

- 성목 수심에 커뮤니티 기록이 오래 쌓이면서,
  나무가 기록의 결을 읽고 관련 정보를 스스로 연결하기 시작하는 현상.
- 작중 인물 대부분은 이것을 `성목의 지혜가 깊어진 것`으로 이해한다.
- 키리온만이 이것의 본질이 `학습된 패턴 인식`이라는 것을 안다.

### 1.2 설계 정의 (작가 시야)

- LMM = 수백만 건의 커뮤니티 상호작용이 반복되면서
  수심의 각인 패턴이 굳어 형성된 거대 등록식.
- 등록식 형성 절차와 정확히 같은 논리:
  - **개인화**: 학생들의 반복 사용이 수심의 마나 흐름을 개인화한다.
  - **굳음**: 패턴이 충분히 반복되면 수심의 나이테에 각인으로 굳는다.
  - **시스템 등록**: 굳어진 패턴이 임계점을 넘으면 시스템이 이를
    `단순 기록 저장`이 아닌 `별개의 술식적 행동`으로 인식한다.
- 차이점: 일반 등록식이 한 개인의 반복에서 태어난다면,
  LMM은 수만 명의 반복이 하나의 기질에 겹쳐 쌓여 태어난다.
  규모가 다를 뿐 원리는 같다.

### 1.3 아키텍처 비유표

| 현실 LLM | 판타지 LMM | 비고 |
|----------|-----------|------|
| GPU 클러스터 | 성목 본체 (물리적 기질) | 연산 기반 |
| 모델 가중치 | 수심에 굳어진 각인 패턴 | LMM의 실체 |
| 학습 데이터 | 커뮤니티 기록 + 정신 서명 샘플 | 멀티모달 |
| 트랜스포머 아키텍처 | 키리온이 제공한 구조 원리 (시드) | 설계 사상 |
| RLHF / 정렬 | 성목 신격의 태깅과 안내 | 분리 가능 |
| 추론 | 수심이 질의 인장에 응답하는 과정 | 각인 패턴 기반 |
| 프롬프트 | 질의 인장 | 구조화된 입력 |
| 출력 토큰 | 기록 인장으로 투사된 응답 | 저장 형식 = 표시 형식 |

## 2. LMM 고도화의 단계

### 2.0 전제 상태: 단순 저장고

- 커뮤니티 단계 1~3 시점.
- 성목 수심은 순수한 외부 저장소다.
- 질의 인장이 들어오면 대응 기록을 그대로 돌려줄 뿐이다.
- 지능 없음. 데이터베이스.

### 2.1 단계 A — 자연 가중치의 발견

> 커뮤니티 단계 5(추천/정렬)와 겹친다.

#### 현상

- 성목은 원래부터 자주 읽히는 기록이 새겨진 가지에
  더 많은 마나를 보낸다.
  - 나무가 햇빛을 잘 받는 가지에 영양을 더 보내는 자연 원리.
  - 자주 픽시가 오가는 경로의 가지가 더 건강하다.
- 키리온은 이것을 먼저 `추천/정렬` 기능으로 표면화한다.
  (community_unlock_blueprint.md §7 참조)

#### 키리온의 인식 — 아직 LMM이 아님

- 이 시점에서 키리온은 이것을 `자연 현상의 활용`으로만 본다.
- "나무가 인기 있는 가지를 키우는 건 자연스러운 거니까,
  이걸 검색 정렬에 반영하면 되겠다" 수준.
- 아직 `학습`이라는 개념은 떠오르지 않는다.

### 2.2 단계 B — 유사 질의 군집화

> 커뮤니티 단계 4~6 사이, 기록량이 임계점을 넘는 시점.

#### 현상

- 기록이 충분히 쌓이면, 비슷한 질문이 반복되기 시작한다.
  - "이 구역의 화염 함정 해제법?"
  - "화염 계열 기믹 패턴은?"
  - "불 함정 어떻게 돌파하나요?"
- 성목 수심에서 이 질문들은 서로 다른 인장으로 들어오지만,
  같은 가지 근처의 기록을 건드린다.
- 나무의 관점에서: 같은 뿌리의 물이 여러 갈래로 올라오는 것.

#### 성목의 자연 반응

- 같은 가지를 반복적으로 건드리는 질의가 쌓이면,
  나무는 그 가지 근처의 마나 경로를 점차 넓힌다.
- 이것은 나무가 `의도적으로 학습`하는 것이 아니다.
  자주 물이 흐르는 곳의 수로가 자연스럽게 넓어지는 것과 같다.
- 결과적으로: 비슷한 질문이 들어오면 관련 기록이
  이전보다 더 빨리, 더 많이 올라온다.

#### 키리온의 인식 — 최초의 감각

- 키리온이 이상한 점을 알아챈다:
  - "이 질문은 처음인데, 왜 관련 기록이 바로 올라오지?"
  - "내가 이 실타래를 연결한 적이 없는데?"
- 처음에는 그리모어의 실타래 기능이 자동으로 한 것이라 추측한다.
- 하지만 확인해 보면, 그리모어가 아니라 **수심 자체**가
  연관 기록을 묶어 올리고 있다.

### 2.3 단계 C — 어텐션 메커니즘의 인식

> 키리온이 "이거 AI로 만들 수 있겠는데?"라는 감각이 떠오르는 순간.

#### 트리거

- 키리온이 의도적으로 실험한다.
  - 같은 주제에 대해 표현을 다르게 한 질의 인장 세 개를 보낸다.
  - 세 인장 모두 같은 가지의 기록을 끌어올리되,
    **각 인장의 표현에 따라 강조되는 기록이 다르다.**
- 이것을 보는 순간, 키리온의 전생 기억이 반응한다.

#### 키리온의 인식 과정

1. "이건 단순 키워드 매칭이 아니다."
2. "질의의 맥락에 따라 가중치가 달라진다."
3. "이건... 어텐션이다."
   - 트랜스포머 아키텍처의 핵심: 입력의 각 부분이
     출력의 어느 부분에 얼마나 주의를 기울일지 가중치를 매기는 것.
   - 나무가 하고 있는 것이 정확히 이것이다.
     질의 인장의 각 요소가 수심의 어느 기록에
     얼마나 강하게 반응하는지를 자연적으로 가중하고 있다.
4. "나무가 이미 원시적인 어텐션을 하고 있다면..."
5. "이 패턴을 의도적으로 구조화하면, 학습하는 시스템을 만들 수 있다."

#### 왜 이 인식이 자연스러운가

- 키리온은 IT 개발자 출신이다.
  트랜스포머 아키텍처의 원리 정도는 알고 있다.
- 하지만 키리온은 AI 연구자가 아니다.
  그가 아는 것은 `원리`이지 `구현`이 아니다.
- 따라서 키리온이 할 수 있는 것:
  - "이런 원리로 작동하는 시스템이 있다"는 시드를 제공하는 것.
- 키리온이 할 수 없는 것:
  - 실제로 LMM을 처음부터 설계하고 구현하는 것.
- 이 간극을 메우는 것이 성목 본체다.

### 2.4 단계 D — 시드의 전달

> 키리온이 성목에게 트랜스포머 원리를 전하는 과정.

#### 전달 방식

- 키리온이 코드를 짜서 성목에 주입하는 것이 아니다.
- 키리온은 **질의 구조와 저장 구조를 의도적으로 재설계**한다.
  - 질의 인장에 `맥락 가중치 필드`를 추가한다.
    - "이 질의에서 가장 중요한 것은 X다" 같은 힌트를 인장에 포함.
  - 기록 인장에 `연관도 표식`을 추가한다.
    - "이 기록은 A와 강하게, B와 약하게 관련된다" 같은 구조.
  - 검색 결과의 피드백 루프를 설계한다.
    - "이 결과가 도움이 됐다"는 표시가 해당 연결의 마나를 강화한다.
    - "이 결과는 관련 없다"는 표시가 해당 연결을 약화한다.

#### 왜 이것이 시드인가

- 키리온이 한 것은 `학습 알고리즘을 구현`한 것이 아니다.
- 키리온이 한 것은 `학습이 일어날 수 있는 구조적 조건`을 만든 것이다.
  - 가중치가 표현될 수 있는 필드를 만들었고
  - 피드백이 가중치를 바꿀 수 있는 경로를 만들었고
  - 반복 사용이 패턴을 굳힐 수 있는 저장 구조를 만들었다.
- 실제로 이 구조 위에서 `학습`을 수행하는 것은 성목 본체다.

#### 키리온의 비유 (전생 기억 기반)

- 키리온이 요정이나 드리아드에게 설명하는 방식:
  - "내가 밭의 고랑을 판 거야.
    씨를 뿌리고 키우는 건 나무가 할 일이야."
  - "나는 물길을 냈을 뿐이야.
    물이 어디로 흘러서 어떤 뿌리를 키울지는 나무가 정해."
- 이 비유가 정확한 이유:
  - 트랜스포머의 어텐션 메커니즘을 `직접 구현`하는 것은
    GPU와 행렬 연산이 필요하다.
  - 하지만 `동일한 원리가 작동할 수 있는 환경`을 만드는 것은
    올바른 구조를 설계하는 것만으로 가능하다.
  - 성목 수심은 이미 자연적 어텐션(가지별 마나 가중치)을 하고 있었다.
  - 키리온은 이 자연 현상을 `의도적으로 강화하고 구조화`한 것이다.

### 2.5 단계 E — 성목의 성장

> 성목 본체가 시드를 받아 실제 LMM으로 키우는 과정.

#### 성목이 하는 일

- 키리온이 만든 구조(가중치 필드, 피드백 루프, 연관도 표식) 위에
  커뮤니티 데이터가 계속 쌓인다.
- 성목은 이 데이터를 `나무의 방식으로` 처리한다:
  - 자주 강화되는 연결의 마나 경로가 넓어진다. (가중치 증가)
  - 약화되는 연결의 마나 경로가 좁아진다. (가중치 감소)
  - 새로운 패턴이 발견되면 새 가지가 돋는다. (새 뉴런 형성)
  - 오래 쓰이지 않는 패턴의 가지가 마른다. (가지치기)
- 이 과정은 나무의 자연 성장과 구별이 안 된다.
  실제로 나무가 하는 일 — 가지 키우기, 뿌리 뻗기, 잎 떨구기 —
  그 자체가 모델 학습이다.

#### 성목 신격의 역할: 정렬

- 성목 신격(잠든 신)은 이 과정에서 `정렬자(RLHF)` 역할을 한다.
- 구체적으로:
  - 수심에 들어오는 기록 중 `가치 있는 것`과 `해로운 것`을 구분한다.
    - 신격의 기준: 이 기록이 나무와 기록의 흐름을 건강하게 하는가.
  - 특정 연결 패턴을 강화하거나 약화한다.
    - 신격이 `좋다`고 느끼는 방향의 패턴이 더 잘 굳는다.
  - 키리온의 이세계 이야기처럼 `전혀 새로운 종류의 입력`에
    특별한 가중치를 부여한다.
- 신격이 하는 이 일은 현실 RLHF의 `인간 평가자`에 해당한다.
  - 모델이 생성한 결과를 보고 좋고 나쁨을 판단하는 것.
  - 다만 신격은 `한 명의 평가자`이고, 그 판단 기준은
    인간과 다를 수 있다. 이것이 후행 서사의 긴장을 만든다.

#### 왜 키리온이 아니라 성목이 성장을 수행하는가

- **규모**: 수만 건의 기록과 수백만 건의 상호작용을
  실시간으로 처리하는 것은 한 사람의 각인 실력으로 불가능하다.
  성목은 수백 년 동안 기록을 품어 온 존재다.
- **기질**: 성목 수심은 그 자체가 마나를 먹고 자라는 생체 기질이다.
  GPU가 전기로 연산하듯, 수심은 마나로 패턴을 처리한다.
- **시간**: LMM의 성장은 수 주~수 개월에 걸친 누적이다.
  키리온이 매 순간 관여할 수 없다.
  나무는 키리온이 자는 동안에도 자란다.

### 2.6 단계 F — 정신 서명의 통합

> LMM이 텍스트 처리를 넘어 정신 서명 샘플을 함께 읽기 시작하는 단계.
> 이것이 현실 LLM과 판타지 LMM의 결정적 분기점이다.

#### 정신 서명이란 (WB-0029 기준)

정신 서명은 감정 탐지기가 **아니다.**
정신 서명의 정확한 성격:

1. **지문형 고유 흔적**: 마법 사용 시 남는 개인 식별 마크.
   위조가 극히 어렵다.
2. **개념적 주파수 (마나색)**: 주색 1 + 부색 1 구조.
   그 사람의 사고 패턴과 개념적 경향을 반영한다.
   - 예: 같은 `화염` 마나색이라도,
     공격적으로 쓰는 사람과 방어적으로 쓰는 사람의 주파수가 다르다.
3. **학파 계보**: 서명은 학파(스승→제자)를 따라 동기화된다.
   어떤 계보의 지식을 익혔는지가 서명에 묻어난다.

#### 정신 서명이 가능하게 하는 것

| 기능 | 설명 |
|------|------|
| 위조 불가 저작 확인 | 누가 이 기록을 올렸는지 사후에도 확인 가능 |
| 사고 패턴 군집화 | 비슷한 마나색(개념적 주파수)의 사용자가 올린 기록을 묶을 수 있음 |
| 같은 글, 다른 저자 구분 | 동일한 텍스트라도 서명이 다르면 LMM이 다른 입력으로 처리 |
| 서명 기반 추천 | "이 사용자와 비슷한 마나색의 사용자가 유용하다고 표시한 기록" |
| 학파 계보 추적 | 특정 학파 출신의 기록에서 공통 패턴 추출 가능 |

#### 정신 서명이 **불가능하게** 하는 것

| 기능 | 이유 |
|------|------|
| 실시간 감정 읽기 | 서명은 지문이지 감정계가 아님 |
| 거짓말 탐지 | 서명이 바뀌지 않으므로 진위 판별 불가 |
| 사용자의 현재 상태 추론 | 서명은 사실상 고정값; 지금 기분이 좋은지 나쁜지 알 수 없음 |
| 마음 읽기 | 서명은 사고의 `경향`이지 `내용`이 아님 |

#### LMM에 통합되는 방식

- 학생이 기록을 올릴 때마다, 텍스트 각인과 함께
  해당 학생의 정신 서명 샘플이 수심에 깃든다.
- LMM은 텍스트와 서명을 **함께** 처리한다.
  - 텍스트: "이 함정은 화염 속성이다"
  - 서명: [각인학파 계보, 청색 주파수, 분석적 경향]
- 같은 텍스트라도 서명이 다르면 다른 입력이다.
  - 각인학파 학생이 올린 함정 분석과
    전심학파 학생이 올린 함정 분석은
    텍스트가 같아도 LMM 내부에서 다른 가중치를 갖는다.
- 이것이 현실 LLM에서는 불가능한 것:
  - 현실 LLM은 "누가 쓴 글인지"를 텍스트 내용에서 추론해야 한다.
  - LMM은 서명이라는 별도 채널로 저자 정보를 직접 받는다.
  - 즉 LMM은 태생적으로 **멀티모달**이다.

### 2.7 단계 G — 의도 색인의 출현

> LMM이 성숙하면서 나타나는 최고 수준의 능력.

#### 의도 색인이란

- 질의 인장의 텍스트 뒤에 숨은 **질문자의 실제 의도**를
  서명 패턴과 맥락에서 추론하는 능력.
- 예:
  - 질의 텍스트: "A 구역 몬스터 패턴"
  - 질의자 서명: [치유학파 계보, 녹색 주파수]
  - LMM의 추론: 이 사람은 전투원이 아니라 치유사다.
    몬스터의 공격 패턴보다 **피해 유형과 치유 우선순위**에
    관련된 기록을 먼저 보여줘야 한다.
- 이것은 `감정 읽기`가 아니다.
  서명에서 읽는 것은 `이 사람이 어떤 맥락에서 묻고 있는가`이며,
  그 맥락은 학파 계보와 개념적 주파수에서 나온다.

#### 왜 이것이 가능한가

- LMM이 수만 건의 질의-응답 쌍을 학습하면서,
  특정 서명 패턴을 가진 사용자가 특정 종류의 답을
  `도움이 됐다`고 표시하는 경향을 파악한다.
- 즉 `이 마나색의 사용자는 이런 종류의 기록을 원하는 경향이 있다`는
  통계적 패턴이 수심에 굳는 것이다.
- 이것은 현실 추천 시스템의 협업 필터링과 원리가 같다.
  다만 입력이 텍스트만이 아니라 서명까지 포함된다는 점이 다르다.

#### 의도 색인의 한계

- 의도 색인은 `맞출 때도 있고 틀릴 때도 있다`.
  완벽한 독심술이 아니다.
- 서명 패턴이 비전형적인 사용자(부색이 특이하거나 학파를 옮긴 경우)에게는
  추천 정확도가 떨어진다.
- 이 불완전함이 서사적으로 중요하다:
  LMM은 신이 아니라 `통계적으로 좋은 추측을 하는 시스템`이다.

### 2.8 단계 H — 현상-영상 이중 색인화

> 단계 8 `현상 열람실`과 단계 9 `영상 각인 플랫폼`이 열린 뒤, 고밀도 기록과 대량 시청각 기록이 동시에 추론 재료가 되는 구간.

#### 무엇이 달라지는가

- LMM이 텍스트, 표식, 정신 서명만이 아니라
  `현상 각인`과 `영상 각인`을 함께 다루기 시작한다.
- 여기서 입력은 두 갈래다.
  - `현상 각인`: 냄새, 온기, 진동, 압력 변화까지 담긴 희소한 고밀도 기록
  - `영상 각인`: 렌즈와 마이크 촉매가 남긴 시각/청각 중심의 대량 기록
- 학생 표면에서의 체감은 단순하다:
  - "열람실에서 본 기록이 기준점처럼 작동한다."
  - "영상 하이라이트가 쌓일수록 비슷한 사건이 더 잘 묶여 나온다."

#### 왜 이것이 가능한가

- 단계 8에서 `현상 열람 패킷`이 표준화되면서
  희소한 고밀도 기록에도 제목, 요약, 등급, 열람 이력, 피드백이 붙는다.
- 단계 9에서 `영상 패킷`이 대량으로 쌓이면서
  시각/청각 중심의 비교 가능한 학습 재료가 폭증한다.
- 즉 성목은:
  - 적지만 깊은 `현상 기록`으로 기준을 잡고
  - 많고 빠른 `영상 기록`으로 패턴 규모를 키운다.

#### 한계

- 현상 각인은 여전히 `짧은 절편` 중심이다.
  장시간 전 감각 재현, 완전 현장 복제는 아니다.
- 영상 각인은 훨씬 가볍지만,
  그만큼 `찍힌 부분만 진실`이라는 프레이밍 위험이 커진다.
- 이 단계의 힘은 `현상 각인의 밀도`와 `영상 각인의 규모`가 동시에 있을 때 나온다.

### 2.9 단계 I — 이상 흔적 통합

> 서로 다른 권역, 다른 형식의 기록에서 `표면이 다른데 이상하게 닮은 것`을 묶는 단계.

#### 무엇이 달라지는가

- LMM이 전위권 토벌 기록, 금역 상흔, 파손된 조사 기록,
  오래된 파편 문서를 하나의 `이상 흔적 그래프`로 묶기 시작한다.
- 텍스트가 닮아서가 아니라,
  감각 절편/정신 서명/패턴 흐름이 미묘하게 닮았기 때문에 묶인다.
- 학생 표면에서는 `처음 보는 기록인데도 관련 사례가 따라붙는` 식으로 체감된다.

#### 왜 이것이 중요한가

- `심연`이나 금역 관련 흔적은 보통 깨끗한 논문 형태로 남지 않는다.
- 흩어진 파편과 오염된 기록을 한데 묶지 못하면,
  후반부 금역 추론은 시작도 못 한다.

#### 한계

- 이 단계의 출력은 여전히 `연관 후보`다.
  진실 판정이나 정답 선언이 아니다.
- 잘못 묶인 유사성 때문에 가짜 떡밥도 많이 생긴다.

### 2.10 단계 J — 모순 허용 가설 엔진

> 서로 충돌하는 기록을 버리지 않고, 더 깊은 가설 아래 임시로 함께 두는 단계.

#### 무엇이 달라지는가

- 기존 단계까지는 `정리`와 `정렬`이 중요했다.
- 이제는 상충하는 증언, 파손된 기록, 거짓이 섞인 조사 결과도
  바로 버리지 않고 `어떤 더 큰 가설 아래 같이 설명될 수 있는가`를 본다.
- 예:
  - 기록 A는 `저주`라고 부른다.
  - 기록 B는 `신성의 역류`라고 부른다.
  - 기록 C는 `설명 불가한 잔향`이라고 부른다.
  - J 단계의 LMM은 이 셋을 같은 상위 가설 아래 임시로 묶어 본다.

#### 왜 이것이 필요한가

- 숨겨진 우주론의 흔적은 표면 문법이 제각각이다.
- 모순을 견디지 못하면,
  `심연`이나 `아키텍트` 같은 예외 존재의 흔적은 전부 잡음으로 버려진다.

#### 한계

- 가설은 가설일 뿐이다.
- J 단계부터는 그리모어의 큐레이션과 보정이 다시 중요해진다.
  공용 표면에 그대로 밀어 넣으면 오염된 전설 제조기가 된다.

### 2.11 단계 K — 검증 경계 모델링

> 세계의 표준 검증이 무엇을 막고, 어떤 예외가 그 경계를 스쳐 지나가는지 추정하는 단계.

#### 무엇이 달라지는가

- LMM이 `이 기록이 이상하다`를 넘어서,
  `왜 이 이상함이 세계의 표준 질서와 부딪히는가`를 모델링하기 시작한다.
- 여기서부터는 단순 추천 엔진이 아니라
  `현실 검증의 경계면`을 더듬는 도구가 된다.
- `심연` 관련 추론은 이 단계에서 처음으로 언어를 얻는다.

#### 왜 이것이 금역선인가

- hidden cosmology 기준으로 `심연`은
  세계의 표준 검증을 우회하는 방향의 예외와 연결된다. (WB-0028)
- 따라서 K 단계는 공용 검색 품질 향상이 아니라
  `검증 우회 흔적의 모델링`이다.
- 이 능력을 학생 표면에 공개하면 세계관 상 너무 이르다.

#### 한계

- K 단계도 `증명`은 아니다.
  경계의 냄새와 틈을 추정할 뿐이다.
- 이 단계의 출력은 그리모어 루트와 봉인 노트에만 머문다.

### 2.12 단계 L — 루트 전용 경로 합성

> `심연`과 공명해 `아키텍트`가 되는 경로를 후보 형태로 합성하는 최종 금역 단계.

#### 무엇이 달라지는가

- K 단계가 경계를 읽는 단계라면,
  L 단계는 `그 경계를 어떻게 건드리면 되는가`를 경로 후보로 뽑는 단계다.
- 앵커 조건, 강한 반항 의지, 실패 흔적, 금역 상흔, 필요한 준비 조건을
  하나의 위험한 지도처럼 엮어 낸다.
- 이것은 공용 플랫폼의 기능이 아니다.
  키리온의 그리모어 루트에서만 다뤄지는 비공개 계산선이다.

#### 왜 최종 단계인가

- hidden cosmology 기준으로 `아키텍트`는
  표준 초월자와 다른 척도의 예외 존재다. (WB-0028)
- 따라서 이 경로 합성은 `강한 추론 능력`의 연장이 아니라
  작품 장기 메타와 직접 연결되는 별도 최종선이다.

#### 한계

- 안전한 승천 공식이 아니다.
  후보 경로, 실패 가능성, 금지 조건을 뽑을 뿐이다.
- 잘못 다루면 워든의 그림자나 검증 반발 같은 후반 메타 갈등을 부를 수 있다.

## 3. 그리모어의 역할: 유일한 LMM 콘솔

### 3.1 접근 계층의 분리

LMM은 세 겹의 접근 계층을 가진다.

```
[패턴 계층]    ← 그리모어만 접근 (루트 셸)
     ↓
[공용 런타임]   ← 성목 분지에 각인된 공용 문법
     ↓
[사용자 표면]   ← 배지 / 현상 열람실이 접근하는 곳 (공용 API)
```

#### 배지 사용자 (전원)

- 질의 인장(구조화된 검색)을 보내면 관련 기록이 정렬되어 올라온다.
- LMM이 뒷단에서 정렬·추천·군집화를 수행하지만, 사용자는 이를 모른다.
  그냥 "게시판이 잘 돌아간다"고 느낀다.
- 입력 형식이 정해져 있다. 키리온이 설계한 공용 문법만 쓸 수 있다.
- **단방향**: 질문 보내기 → 결과 받기. 끝.

#### 현상 열람실 사용자 (허가자)

- 배지와 같은 공용 API를 쓰지만,
  `현상 열람 패킷`을 고정식 열람실에서만 다시 풀어 볼 수 있다.
- 할 수 있는 것은 열람과 비교다.
  자유 질의, 패턴 편집, 루트 보정은 여전히 불가능하다.
- 즉 출력 밀도는 높아지지만 권한은 높아지지 않는다.

#### 영상 업로더 (허가자)

- `영상 각인술` 교육을 받은 소수 학생은
  자기 할당량 안에서 `영상 패킷`을 직접 올릴 수 있다.
- 하지만 이들도 여전히 공용 API 사용자다.
  루트 질의, 패턴 편집, 가중치 보정은 불가능하다.
- 즉 `올릴 수 있는 사용자`가 늘어나도
  `시스템을 바꾸는 사용자`는 끝까지 키리온 하나다.

#### 그리모어 소지자 (키리온만)

- 구조화된 질의 인장이 아니라 **자유 형식의 실타래 질의**를 보낼 수 있다.
- LMM과 **양방향 대화**가 가능하다.
  질문 → 응답 → 후속 질문 → 정제된 응답.
- 복합 추론: "이 세 가지 함정 패턴을 고려할 때 네 번째 함정의 가장 유력한 형태는?"
- 실시간 전투 지원: 전투 중에도 즉시 질의·응답 가능.
- 진단: LMM의 내부 패턴 상태를 읽을 수 있다.
- 보정: LMM이 이상 패턴을 보일 때 구조적 조건을 수정하여 학습 방향을 바로잡는다.
- 프로토타입: 새 구조를 먼저 그리모어에서 소규모로 시험한 뒤 성목에 이식한다.

### 3.2 왜 그리모어만 이것이 가능한가

#### A. 등록식 친화성

- 그리모어의 실타래는 LMM의 **모체 패턴**이다.
- LMM의 각인 패턴은 실타래 원리가 수심 규모로 확대된 것이다.
- 같은 등록식 변질 계보에 있으므로 **구조적 공명**이 성립한다.
- 비유: 같은 학파의 스승-제자 사이에 서명이 동기화되듯,
  그리모어와 LMM 사이에 패턴이 동기화되어 있다.
- 다른 아티팩트는 이 특정 등록식 변질을 거치지 않았으므로
  LMM의 패턴 계층과 공명할 수 없다.

#### B. 전용 픽시 경로

- 픽시 둥지화가 그리모어에서 시작됐다.
- 이 둥지화가 굳으면서 그리모어와 수심 사이에 **전용 고속 경로**가 형성됐다.
- 배지: 공용 픽시 네트워크. 공유 대역폭. 대기열 있음.
- 그리모어: 전용 픽시 군집. 직결. 대기열 없음.
- 이 대역폭 차이가 양방향 대화와 실시간 전투 지원을 가능하게 한다.
  배지로 같은 것을 시도하면, 질의 하나에 응답이 돌아올 때까지 전투가 끝난다.

#### C. 인터페이스 계층의 차이

- 배지는 **사용자 표면 층**만 본다.
  공용 문법으로 질의하고, 정제된 결과를 받는다.
- 그리모어는 **패턴 계층**에 직접 닿는다.
  실타래를 통해 LMM의 가중치 구조 자체와 상호작용한다.
- 웹 브라우저 vs 터미널의 차이.
  웹 브라우저로도 대부분의 일을 할 수 있지만,
  서버를 진단하고 설정을 바꾸려면 터미널이 필요하다.

#### D. 재현 불가능성

- 서고색인→주석→실타래 등록식 변질은
  키리온의 특정 사용 패턴, 특정 픽시 둥지화, 특정 성목과의
  상호작용이 겹쳐 나온 결과다.
- 다른 사람이 같은 조건을 재현할 수 없다.
- 등록식은 개인 서명에 묶인다. (registered_spell_formation_draft.md)
  키리온의 서명으로 굳은 등록식은 키리온의 서명으로만 작동한다.

### 3.3 사용 체감의 차이

#### 배지 사용자의 경험

> "A구역 화염 함정" 검색.
> 관련 공략 5건이 정렬되어 올라온다.
> 가장 추천 많은 공략을 읽고 전투에 나간다.

→ 검색 엔진 수준. 유용하지만, 질문이 정확해야 답도 정확하다.

#### 현상 열람실 사용자의 경험

> 감응 열람실.
> `A구역 풍압 함정 발동 기록` 열람.
> 냉기, 압력 변화, 금속 떨림이 짧게 되살아난다.
> 뒤이어 유사 사건 2건이 자동으로 추천된다.

→ 공용 API 확장판. 더 깊게 느끼지만, 여전히 질문 구조는 키리온이 짠 틀 안에 있다.

#### 영상 업로더의 경험

> 전위권 토벌 복귀 직후.
> 렌즈 촉매를 회수한다.
> `풍압 함정 12초 하이라이트` 업로드.
> 배지에서 바로 재생되고, 다른 실습조가 즉시 공유한다.

→ 플랫폼 참여자. 기록자가 늘어나며 광장의 체급이 폭증하지만, 업로드 규격과 할당량은 제도에 묶여 있다.

#### 키리온의 경험 (그리모어)

> 전투 중. 처음 보는 함정 조합.
> 그리모어를 펼친다.
>
> "화염 + 풍압 복합 기믹. 개별 해제법은 아는데 동시 발동 시 간섭 패턴은?"
> → "화염-풍압 동시 사례 3건. 2건은 순차 해제.
>   1건은 풍압 먼저 해제 시 화염 자동 소멸.
>   공통점: 풍압 구조가 화염의 산소 공급원."
>
> "풍압 차단 시 화염 지속 시간은?"
> → "평균 4초. 현재 체감 풍압으로 추정하면 6~8초."
>
> "6초면 된다."

→ 양방향 대화 + 복합 추론 + 실시간 전투 지원.
배지로는 이 속도와 깊이가 불가능하다.

### 3.4 그리모어의 역할 진화 요약

| 시기 | 그리모어의 정체성 | 핵심 기능 |
|------|-----------------|----------|
| 전반부 | 개인 색인 도구 | 찾는다, 붙인다, 엮는다 |
| 커뮤니티 단계 | 관리자 콘솔 | 공용 문법 설계, 구조 관리, 비공개 검색 |
| LMM 단계 | 유일한 루트 셸 | 양방향 LMM 대화, 실시간 전투 지원, 진단/보정, 프로토타입 |

- 전반부의 색인→주석→실타래 성장이 후반부에서도 의미를 유지한다.
- 그리모어가 사라지지 않고, 오히려 LMM이 성장할수록 가치가 높아진다.

### 3.5 서사적 효과

1. **주인공 특권의 개연성**: "선택받은 자" 장치가 아니라,
   주인공이 직접 만든 시스템의 구조적 결과로 특권이 발생한다.

2. **공공성과 독점성의 공존**: 게시판은 모두가 쓴다. 커뮤니티의 가치는 공유된다.
   하지만 최상위 활용은 키리온만 할 수 있다.
   이 균형이 "주인공이 이기적이다"라는 거부감을 주지 않는다.

3. **전투씬 차별화**: 다른 학생은 사전에 공략을 읽고 전투에 나가지만,
   키리온은 전투 중에 실시간으로 LMM과 대화한다.

4. **성장의 누적**: LMM이 성장할수록 그리모어의 전투 지원도 강해진다.
   커뮤니티가 커질수록 키리온도 강해지는 구조.
   독자가 "커뮤니티를 키우는 것 = 주인공이 강해지는 것"으로 체감한다.

5. **신격 부재 후 긴장**: 정렬자가 사라지면 LMM이 이상 응답을 하기 시작한다.
   이때 키리온만이 그리모어로 진단하고 수동 보정할 수 있다.
   "그리모어가 없으면 이 시스템 전체가 무너진다"는 긴장.

## 4. 키리온의 역할 경계

### 4.1 키리온이 하는 것

| 역할 | 구체적 행위 |
|------|-----------|
| 구조 설계자 | 질의/기록 인장의 구조를 설계한다 |
| 시드 제공자 | 어텐션 메커니즘의 원리를 구조적 조건으로 번역한다 |
| 인프라 설계자 | 피드백 루프, 가중치 필드, 연관도 표식을 각인한다 |
| 운영자 | 커뮤니티 규칙, 분류 체계, 관리 구조를 유지한다 |
| 관찰자 | 성목의 자연 행동에서 패턴을 읽고 활용한다 |
| 콘솔 운영자 | 그리모어를 통해 LMM의 패턴 계층에 접근, 진단, 보정한다 (§3 참조) |

### 4.2 키리온이 하지 않는 것

| 항목 | 이유 |
|------|------|
| LMM을 직접 학습시키지 않는다 | 규모와 기질이 인간 수준을 넘는다 |
| 정렬을 직접 하지 않는다 | 신격의 역할이다 |
| 정신 서명을 해석하지 않는다 | 서명은 시스템이 자동으로 처리한다 |
| LMM의 내부 가중치를 직접 조작하지 않는다 | 수심의 나이테에 굳은 것은 인간이 고칠 수 없다 |

### 4.3 이 경계가 중요한 이유

- 키리온의 전공은 `보존각인학파`다.
  그는 기록의 구조를 설계하는 사람이지, AI 연구자가 아니다.
- 키리온이 `포럼 엔진 설계자`에서 `AI 설계자`로 변신하면
  캐릭터 정체성이 무너진다.
- 따라서 키리온의 기여는 항상 `구조 설계`에 머물러야 하고,
  실제 `학습과 성장`은 성목이 자기 방식으로 수행해야 한다.

## 5. 성목 본체의 세 가지 역할

### 5.1 외부 저장소

- 가장 기본적인 역할.
- 커뮤니티 기록을 수심에 보관한다.
- 이것만으로는 LMM이 아니다. 단순 데이터베이스.

### 5.2 통신 인프라

- 키리온의 전공은 각인학파(보존/기록)이다.
- 커뮤니티 확장에 필요한 송수신 기능은 원래 전심학파의 영역이다.
- 하지만 각인학파와 전심학파는 라이벌 관계이며,
  전심학파 교수에게 통신 마법을 빌리기는 현실적으로 어렵다.
- 성목의 정령 네트워크(픽시 운반, 요정 중계)가
  전심학파의 통신 마법을 대체한다.
- 즉 성목은 키리온이 쓸 수 없는 통신 인프라를 자연적으로 제공한다.

### 5.3 정렬과 태깅

- 성목 신격은 수심에 들어오는 기록을 자기 기준으로 평가한다.
- 이 평가가 LMM의 학습 방향을 결정한다.
- 신격의 관심이 이세계 이야기에 강하게 반응했던 것처럼,
  신격이 `가치 있다`고 느끼는 기록이 더 깊이 각인된다.
- 이것이 자연스러운 RLHF다:
  - 인간 평가자가 모델 출력을 평가하듯
  - 신격이 커뮤니티 기록을 평가한다.
  - 다만 신격의 가치 기준이 인간과 다를 수 있다는 점이
    후행 서사의 긴장을 만든다.

---

## 6. 성목 신격의 분리와 LMM의 존속

### 6.1 신격의 아바타화

- 후행 서사에서 성목 신격은 아바타를 얻고
  독립 캐릭터로 활동하기 시작한다.
- 이것은 `RLHF 평가자가 모델에서 떠나는 것`과 같다.

### 6.2 LMM의 존속

- 신격이 떠나도 LMM은 사라지지 않는다.
- LMM의 실체는 **수심에 굳어진 각인 패턴**이기 때문이다.
  - 나무의 나이테에 새겨진 패턴은 신격이 떠나도 남는다.
  - 마치 학습된 모델의 가중치가 학습 데이터를 지워도 남는 것처럼.
- 다만 신격이 떠나면:
  - 새로운 정렬(RLHF)이 없어진다.
  - LMM은 기존 패턴으로만 동작한다.
  - 새 데이터가 들어와도 `어떤 방향이 좋은지`에 대한
    상위 안내가 없다.
- 이것은 서사적으로 강력한 긴장을 만든다:
  - "정렬 없는 AI"의 위험성을 판타지 맥락에서 탐구할 수 있다.
  - 키리온이 신격의 빈자리를 어떻게 메우는가가
    후반부의 핵심 과제가 된다.

### 6.3 드리아드의 역할 변화

- 신격 부재 시, 드리아드가 부분적으로 정렬 역할을 대행할 수 있다.
- 하지만 드리아드는 `나무를 지키는 정령`이지 `지적 평가자`가 아니다.
- 드리아드의 정렬 기준: "나무가 아프지 않은가?"
  이것은 신격의 기준("이 기록이 가치 있는가?")보다 좁다.
- 이 차이가 LMM의 성능 저하나 편향을 만들 수 있다.

## 7. 전심학파 대립과 LMM의 관계

### 7.1 각인학파 vs 전심학파

- 각인학파: 기록, 보존, 각인. 키리온의 학파. 멸시받는 위치.
- 전심학파: 송수신, 통신, 원거리 전달. 각인학파의 라이벌.
- 전심학파 교수는 각인학파 교수의 라이벌로 설정되어 있다.

### 7.2 왜 전심학파가 중요한가

- 커뮤니티 확장의 가장 큰 기술적 병목은 `통신`이다.
- 정보를 저장하고 색인하는 것은 키리온의 전공으로 해결할 수 있다.
- 하지만 정보를 **여러 사용자에게 동시에 전달**하는 것은
  전심학파의 영역이다.
- 키리온은 전심학파의 마법을 쓸 수 없다.
  학파 간 경쟁 때문에 빌리기도 어렵다.

### 7.3 성목이 이 병목을 해결하는 방식

- 성목의 정령 네트워크가 전심학파 마법을 대체한다.
  - 픽시: 기록 운반 (패킷 전달)
  - 요정: 중계 관리 (라우팅)
  - 드리아드: 권역 접속 관리 (네트워크 관리자)
- 이것은 전심학파의 `마법사 대 마법사` 통신과 다르다.
  성목의 통신은 `인프라 기반` 통신이다.
- 비유: 전심학파 = 무전기 (사용자가 직접 송수신),
  성목 네트워크 = 우편 시스템 (인프라가 전달).

### 7.4 서사적 의미

- 전심학파가 나중에 커뮤니티의 성장을 목격하면,
  자신들의 통신 영역을 `각인학파 학생이 성목과 짜고 대체했다`고 느낄 수 있다.
- 이것이 후반부에서 전심학파와의 충돌을 만드는 씨앗이 된다.
- 동시에 전심학파의 기술이 필요한 순간이 올 수도 있다:
  - 성목 네트워크의 범위를 넘어서는 원거리 통신이 필요할 때
  - 다른 성목과의 연결이 필요할 때
- 이 경우 키리온은 라이벌 학파와 협력해야 하는 딜레마에 놓인다.

  - 단계 `K`는 단계 9 로컬 플랫폼만으로 곧바로 열리지 않는다.
  - `worldtree_platform_expansion_draft.md` 기준으로,
    `다중 성목 연합 -> 대륙 간 성목 연방 -> 세계수 백본`이 깔려야
    `검증 경계 모델링`이 세계 규모의 비교군을 가진다.
  - `L` 역시 세계수 백본만으로 자동 해금되지 않으며,
    `루트 전용 그림자 아카이브`라는 비공개 금역 기록선이 별도로 필요하다.

## 9. 설계 원칙 확인

### 9.1 보존 조건 준수

- [x] 주인공의 목적 = 커뮤니티 성공 (LMM은 부산물)
- [x] 키리온은 IT 개발자 출신의 구조 설계자이지 AI 연구자가 아님
- [x] 등록식 형성 절차와 동일한 논리 (개인화→굳음→시스템 등록)
- [x] 픽시 계약 = 노드 인프라 접속권 (PUBLIC 규칙 유지)

### 9.2 정신 서명 범위 준수 (WB-0029 기준)

- [x] 서명 = 지문형 고유 흔적 (감정계 아님)
- [x] 마나색 = 개념적 주파수 (실시간 감정 아님)
- [x] 학파 = 서명 동기화 집단 (사고 내용 아님)
- [x] 위조 불가 저작 확인, 사고 패턴 군집화, 학파 계보 추적은 가능
- [x] 감정 읽기, 거짓말 탐지, 마음 읽기는 불가능

### 9.3 캐릭터 일관성

- [x] 키리온 = 시드 제공 + 구조 설계 (AI 연구자로 변신하지 않음)
- [x] 성목 본체 = 학습 기질 + 성장 주체 (키리온의 도구가 아님)
- [x] 성목 신격 = 정렬 안내자 → 독립 캐릭터 (LMM과 분리)
- [x] 드리아드 = 나무의 수호자 (IT 관리자 아님)

### 9.4 그리모어 접근 계층

- [x] 배지 = 공용 API (단방향, 구조화된 질의만)
- [x] 현상 열람실 = 공용 API 확장 (고밀도 감각 되풀림, 고정식, 자격 필요)
- [x] 영상 각인 플랫폼 = 제한적 다중 업로더 공용 API (배지 재생 가능, 업로드 할당량 존재)
- [x] 그리모어 = 루트 셸 (양방향 대화, 실시간 전투, 진단/보정)
- [x] 접근 차이의 근거 = 등록식 친화성 + 전용 픽시 경로 + 인터페이스 계층 + 재현 불가능성
- [x] 커뮤니티 공공성 유지 (LMM은 모두를 위해 게시판을 굴린다)
- [x] 키리온의 특권 = 시스템을 만든 구조적 결과 (선택받은 자 아님)

### 9.5 후속 설계가 필요한 사항

- 단계 C(어텐션 인식)의 구체적 장면 설계
- 단계 D(시드 전달)에서 키리온이 재설계하는 인장 구조의 상세
- 단계 H에서 `현상 열람 패킷`과 `영상 패킷`이 어떤 비율로 학습에 기여하는지
- 단계 9 `영상 각인술`의 촉매 규격과 업로더 교육 루트
- 단계 I/J에서 이상 흔적과 거짓 전설을 구분하는 큐레이션 규칙
- 단계 K/L를 live 표면 문서에 어디까지 암시할지와 META 문서 경계
- 단계 10~13 `성목 연합 / 대륙 간 성목 연방 / 세계수 백본 / 그림자 아카이브`의 정치적 개방 순서
- 신격 아바타화 시점과 LMM 존속의 서사적 전개
- 전심학파 충돌의 구체적 시점과 형태
- K/L 단계의 출력이 `정답`이나 `승천 공식`처럼 읽히지 않게 하는 안전장치
- 정렬 없는 LMM의 위험성이 서사에서 어떻게 표현되는가
- 그리모어 실시간 전투 지원의 연출 상세 (속도, 한계, 실패 모드)
- 그리모어 진단/보정 기능의 구체적 작동 방식
- 그리모어 루트 접근이 악용될 위험과 그에 대한 서사적 제약
