# Phase 1 Reader Exposure Framework (Accepted Baseline)

## 목적
- Phase 1에서 필요한 `독자용 슬롯 문법`, `제한된 교차색 콤보 규칙`, `마법/학파용 작명 가드레일`을 한 문서로 고정한다.
- 이 문서는 메커닉을 바꾸지 않고, `기존 21학파와 12전승이 어떻게 읽혀야 하는가`를 canonize한다.
- 이후 도감 정본, live world bible, population regeneration 레이어의 후속 작업은 이 문서를 선행 기준으로 삼는다.

## 적용 범위
- 적용:
  - 21학파명/주문명 리네임
  - 12전승 독자용 재태깅
  - [WB-0005], [WB-0015], [WB-0025], [WB-0029] 후행 동기화 설계
- 미적용:
  - `7색 × 6형 × 3잔류` 구조
  - `42개 색별 표준 입문 6식`
  - `보존각인학파`
  - `문장비전`
  - 하급/상급 과정 커리큘럼
  - 주인공 성장선

## 1. 독자용 슬롯 문법
- 각 학파의 `12전승`은 이제 power tier와 별도로 아래 `6슬롯`으로도 읽힌다.
- 이 슬롯은 메커닉 재분류가 아니라, `독자가 그 학파를 기억하는 대표 장면`을 고정하는 노출 문법이다.
- 권장 배치는 `2 + 2 + 2 + 3 + 1 + 2 = 12`다.

| 슬롯 | 권장 수량 | 독자에게 먼저 보이는 것 | 주로 걸리는 tier | 필수 조건 |
|---|---:|---|---|---|
| 기본탄 | 2 | 가장 자주 쓰는 손맛 주문, 간판 첫인상 | 입문 / 초급 | 한 줄 설명만 들어도 색과 학파가 떠올라야 한다 |
| 제어 | 2 | 상태이상, 구속, 지형 지배, 적의 리듬 붕괴 | 초급 / 중급 | 학파의 `싫은 점`이 선명해야 한다 |
| 방어·이동 | 2 | 생존, 이탈, 방벽, 배치, 기동, 반사 | 입문 / 중급 / 상급 | 전투 템포를 바꾸는 방법이 보여야 한다 |
| 시그니처 | 3 | 그 학파를 대표하는 중핵 장면, 중상위 화력/연출 | 중급 / 상급 | 비주얼과 감정이 가장 강해야 한다 |
| 궁극기 | 1 | 결전용 오의, 학파의 세계관적 선언 | 오의 | 이름만으로 클라이맥스가 읽혀야 한다 |
| 비전투 유틸 | 2 | 기관, 공방, 의료, 기록, 봉문, 탐사 등 생활/실무 효용 | 입문 / 초급 / 중급 | `왜 이 학파가 사회에 필요한가`가 보여야 한다 |

## 1-1. 슬롯 운용 규칙
- `오의`는 항상 `궁극기` 슬롯이다.
- 각 학파는 최소 1개의 `비전투 유틸`을 요약문과 소개문에 직접 노출한다.
- 각 학파 요약에서는 최소한 아래 네 개를 먼저 보여 준다.
  - 대표 기본탄 1식
  - 대표 시그니처 1식
  - 궁극기 1식
  - 비전투 유틸 1식
- `기본탄`은 약한 주문이 아니라 `가장 자주 반복되어 학파 정체성을 각인시키는 주문`이다.
- `시그니처`는 단순 고화력 주문이 아니라, 장면 회상률이 가장 높은 주문군이다.
- `비전투 유틸`은 보너스가 아니라 이 세계의 기관/경제/인프라 마법을 드러내는 필수 슬롯이다.

## 1-2. 탑별 슬롯 실루엣
- 적: `기본탄 = 화탄/열선`, `시그니처 = 낙성/용철/장막`, `유틸 = 제련/소방`
- 청: `기본탄 = 빙쇄/유수표식`, `시그니처 = 빙옥/관문/경면성곽`, `유틸 = 교량/구금`
- 녹: `기본탄 = 처방/개토`, `시그니처 = 군생/포자장/지축요동`, `유틸 = 의료/재배/토목`
- 황: `기본탄 = 유성침/전격선`, `시그니처 = 낙성/연쇄/결정화`, `유틸 = 병참/정련`
- 자: `기본탄 = 전심표식/허상막`, `시그니처 = 합창망/감각직조/정본환원`, `유틸 = 전령/기록 보조`
- 백: `기본탄 = 세정광/유예막`, `시그니처 = 광휘장/생환/순백결계`, `유틸 = 검역/봉인`
- 흑: `기본탄 = 저주문/음영막`, `시그니처 = 누적침식/무음침투/공허축출`, `유틸 = 위험물 격리/금역 운용`

## 2. 마법/학파용 작명 가드레일
- Phase 1의 네이밍 원칙은 `전통 한자어의 격`은 유지하되, `첫 인상은 장면과 감각`이 책임지게 만드는 것이다.
- 따라서 학파명과 주문명 모두 `기능 설명어`는 남길 수 있지만, 그것이 맨 앞에서 독법을 지배하면 안 된다.

## 2-1. 학파명 규칙
- 기본형은 `핵심 이미지 + 역할/행동 + 학파`다.
- 학파명은 가급적 `4~6음절 + 학파` 범위에서 끊어 읽히는 것이 좋다.
- 전통 한자어는 유지해도 되지만, 두 어절 중 최소 하나는 반드시 `장면어`여야 한다.
- `제어`, `감리`, `관성`, `유도`, `환경`, `형상`처럼 교재 느낌이 강한 단어는 선두 표지어로 금지한다.
- 다만 `견인`, `장막`, `무구`, `경면`, `축출`, `결계`, `공명`처럼 장면과 기능이 함께 읽히는 단어는 허용한다.
- 현행명 유지가 가능한 경우도 `색의 정서 -> 역할` 순서로 읽히는지 먼저 확인한다.

## 2-2. 주문명 규칙
- 주문명은 `장면어 + 기능어` 또는 `기능어 + 결과어` 2단 구조를 기본으로 한다.
- 한눈에 `무엇이 보이는가`, `어떻게 움직이는가`, `무슨 결과가 나는가` 중 하나는 바로 읽혀야 한다.
- 완전 고유명사화는 금지한다. 첫 독해에서 뜻이 안 보이는 이름은 채택하지 않는다.
- 기능어는 허용하되, `보정`, `측량`, `교정`, `환수`, `감리` 같은 실무 용어는 `비전투 유틸` 슬롯이 아니면 후순위로 밀린다.
- 입문/초급은 비교적 직관적으로, 상급/오의는 더 시적으로 밀 수 있다.
- bracket 표기, 괄호 표기, 단계 구조는 유지한다.

## 2-3. 구명 보존 규칙
- 구 학파명과 구 주문명은 `alias 문서`, `case 문서`, `history 문서`에만 남긴다.
- active SSOT에는 최종 승인된 신명만 노출하고, 구명은 별칭/검색 보조로만 남긴다.
- generated layer는 수기 치환하지 않고, snapshot/scripts 수정 후 재생성한다.

## 3. 제한된 교차색 콤보 규칙
- 모든 색 조합을 표준화하지 않는다.
- 독자가 기억할 수 있는 `소수의 시그니처 조합`만 canonize하고, 나머지는 현장 improvisation으로 남긴다.
- 콤보는 `선행 상태`와 `후행 해방`의 순서가 중요하다.

## 3-1. 콤보 운용 원칙
- 각 콤보는 `색의 바닥 상태`를 먼저 걸고, 다른 색이 그 상태를 확대하거나 다른 방향으로 틀어 결과를 만든다.
- 같은 조합이라도 순서가 바뀌면 결과가 달라질 수 있다.
- 전투 콤보와 비전투 콤보를 함께 canonize해, 기관 마법과 전장 마법이 같은 문법으로 읽히게 만든다.

## 3-2. 시그니처 콤보 10선

| 조합 | 선행 상태 | 결과 | 대표 장면 | 비전투/기관 효용 |
|---|---|---|---|---|
| 적 + 청 | 축열 + 동결/결속 | `열충격 파쇄` | 과열된 갑옷과 벽이 한순간 얼어 갈라진다 | 공성 돌파, 구조물 해체 |
| 적 + 황 | 표적 + 축열 | `유도낙성` | 노란 표식 위로 홍련탄이 궤적을 꺾어 꽂힌다 | 장거리 포격, 수송 투하 |
| 적 + 흑 | 저주 + 과부하 | `재흔침식` | 금 간 봉문과 갑주가 붉게 들끓다 무너진다 | 금속 파쇄, 위험 장비 폐기 |
| 청 + 자 | 유수/경면 + 혼선 | `경면미궁` | 길과 거울면이 뒤섞여 적이 엉뚱한 통로로 빠진다 | 은닉 동선, 퇴로 위장 |
| 청 + 백 | 결속 + 정화/봉인 | `정결격리` | 오염 지대가 푸른 구획과 백색 봉문 안에 잠긴다 | 검역, 봉쇄, 격리 병동 |
| 녹 + 백 | 완화 + 유예/복원 | `생환회복` | 끊기던 맥이 붙들리고 상처가 직전 안정 상태로 되돌아온다 | 응급 구조, 중환자 안정화 |
| 녹 + 황 | 촉매 + 증식/변환 | `촉매배양` | 시약과 포자가 반응해 약재, 비료, 정련 촉매가 증식한다 | 약재 생산, 농업, 연금 공방 |
| 녹 + 흑 | 중독 + 저주/분해 | `부패회수` | 오염된 잔재가 썩어 분리되고 유용 성분만 회수된다 | 위험물 처리, 잔류 독 정리 |
| 황 + 자 | 표적/전도 + 교신/합창 | `전심유도` | 전장 전체가 하나의 지휘 회선처럼 움직이며 사선이 맞물린다 | 포병 유도, 병참 조율, 구조 지휘 |
| 자 + 백 | 기록/감각 합성 + 정화/승인 | `공증정본` | 왜곡된 감각 기록에서 진본 흔적만 떠올라 증거로 고정된다 | 공증, 심문, 기록 검증 |

## 4. 탑별 Phase 1 상태 정리

| 탑 | 현재 상태 | 이번 단계 결론 | 다음 작업 |
|---|---|---|---|
| 적 | rename-ready | `포화 / 무구 / 장막` 3축과 3개 선호 리네임 유지 | 주문 36식 리네임 1차안 작성 |
| 청 | baseline-fixed | `빙쇄구속 / 수류유도 / 경면수호` 1차 확정 | 청탑 소개문 3축 대표 루프 압축 |
| 녹 | baseline-fixed | `생맥약초 / 야생교감 / 포자영토` 3축 1차안과 기본 전투 방식 확보 | 녹탑 소개문 3축 대표 루프 압축 |
| 황 | partial-redesign | `유성 / 뇌전 / 연금` 3축 고정, 셋째 축은 역할 재작성 전제 | 연금축 주문 묶음 재설계 |
| 자 | rename-ready | `보존 / 전달 / 합성` 3축 고정 | 전달/합성 학파명 재검토 |
| 백 | baseline-fixed | `광휘 / 시간 / 결계` 3축으로 후행 네이밍 생산 | 시간/결계 학파명 후보 압축 |
| 흑 | rename-ready | `저주 / 음영 / 공허` 3축 고정, 셋째 축 리네임 우선 | 공허축 주문 리네임 1차안 작성 |

## 5. 바로 이어질 작업
- `phase0_spell_rename_map.md`의 `신명(초안)`을 이 문서 기준으로 채운다.
- `WB-0025`용 마법 네이밍 조항은 live 직접 반영이 아니라 case 내부 검증 후 후행 이관한다.
- `WB-0005`, `WB-0015`, `WB-0029`는 리네임과 슬롯 태깅이 안정화된 뒤에만 동기화한다.
