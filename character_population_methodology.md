# 캐릭터 모집단 설계 방법론 v1.0

작성일: 2026-03-04

---

## 1. 개요

### 목적
3600명의 캐릭터 모집단을 심리 통계 기반으로 설계한다. 개별 캐릭터 카드를 사전 조형하지 않고, 각 슬롯에 심리 좌표만 부여해 둔다. 실제 캐릭터 조형은 시뮬레이션/집필 시 온디맨드로 수행한다.

### 핵심 원칙
- 사전 조형 최소화: 이름·외형·서사는 시뮬레이션 호출 시 생성
- 좌표 배치만 선행: ID별 심리 수치만 미리 확정
- 중복 방지는 좌표 점유 확인으로 대체 (카드 대 카드 비교 불필요)

---

## 2. ID 체계

```
형식: P-{NNNN}
범위: P-0001 ~ P-3600
```

- 스토리 주인공·핵심 NPC는 별도 CH-* 체계 유지 (현행 유지)
- P-* 슬롯은 모집단 풀. 시뮬에서 소환·조형되면 CH-* ID를 추가 발급

## 3. 빅5 수치 배정

### 3.1 통계 근거

일반 성인 모집단 정규분포 파라미터 (IPIP 기반 메타분석, 0.0~1.0 정규화):

| 특성 | 평균 | 표준편차 |
|---|---|---|
| 개방성 (O) | 0.60 | 0.16 |
| 성실성 (C) | 0.55 | 0.16 |
| 외향성 (E) | 0.50 | 0.18 |
| 우호성 (A) | 0.58 | 0.16 |
| 신경성 (N) | 0.47 | 0.18 |

### 3.2 아르케이온 학술원 보정

왕립학술원 입학자는 일반 모집단 대비 자기선택 편향이 있다:

| 특성 | 보정 방향 | 이유 |
|---|---|---|
| 개방성 (O) | 평균 +0.05 | 마법·학문 지향 선택 |
| 성실성 (C) | 평균 +0.07 | 입학 경쟁 통과 |
| 외향성 (E) | 보정 없음 | 입학 기준과 무관 |
| 우호성 (A) | 보정 없음 | 입학 기준과 무관 |
| 신경성 (N) | 평균 +0.03 | 입시·학업 스트레스 |

### 3.3 배정 방법

각 축을 독립 정규분포에서 샘플링 후 [0.01, 0.99] 클리핑.
5개 축 간 미소 상관은 이 규모에서 무시 가능 (독립 가정).

```
O ~ N(0.65, 0.16²), clip
C ~ N(0.62, 0.16²), clip
E ~ N(0.50, 0.18²), clip
A ~ N(0.58, 0.16²), clip
N ~ N(0.50, 0.18²), clip
```

## 4. 파생 지표 산출

### 4.1 산출 대상

빅5에서 3개 지표를 파생한다. 온라인 탈억제는 성격 특성이 아닌 상황 파라미터이므로 제외(시뮬레이션 실행 시 게시판 설정에서 처리).

| 파생 지표 | 설명 |
|---|---|
| 다크 트라이어드 (DT) | 마키아벨리즘·나르시시즘·사이코패시 복합 경향 |
| 인지 욕구 (NFC) | 복잡한 사고를 스스로 파고드는 내재적 동기 |
| 자기감시 (SM) | 사회적 맥락에 따라 자신을 조율하는 능력/경향 |

### 4.2 파생 공식

빅5 가중합 + 잔차 노이즈. 빅5가 설명하는 분산은 약 25~35%이므로 잔차 분산이 크다 — 같은 빅5 좌표를 가진 인물이라도 파생 지표는 실질적으로 다를 수 있다.

**다크 트라이어드 (DT)**
```
DT_base = -0.50×A - 0.35×C + 0.25×N + 0.20×E - 0.05×O
DT = clip(DT_base + noise(σ=0.15), 0.01, 0.99)
```
- 우호성·성실성이 강한 억제 인자
- 높은 신경성·외향성이 약한 촉진 인자

**인지 욕구 (NFC)**
```
NFC_base = 0.45×O + 0.30×C - 0.15×N + 0.05×E + 0.05×A
NFC = clip(NFC_base + noise(σ=0.12), 0.01, 0.99)
```
- 개방성·성실성이 핵심 예측 인자
- 신경성이 약한 억제 인자

**자기감시 (SM)**
```
SM_base = 0.40×E + 0.15×O + 0.10×A - 0.10×C - 0.20×N
SM = clip(SM_base + noise(σ=0.13), 0.01, 0.99)
```
- 외향성이 핵심 예측 인자
- 신경성이 억제 인자

> **주의**: 상관계수는 메타분석 근사값이며 문화권별 차이가 있다. 정밀도보다 방향성과 상대적 강도가 목적이므로 소수점 2자리 이상 신뢰하지 않는다.

### 4.3 잔차 노이즈 처리

잔차는 빅5로 설명되지 않는 개인차다. LLM 파생 산출 시 시드 값으로 사용할 난수를 미리 생성해 저장해두거나, 생성 시점에 LLM이 자체 판단으로 채우도록 위임한다 (LDD 철학상 후자 권장).

## 5. 산출물 스펙

각 P-* 슬롯의 저장 형식:

```yaml
id: P-0001
b5:
  O: 0.72
  C: 0.58
  E: 0.31
  A: 0.64
  N: 0.55
derived:
  DT: 0.18
  NFC: 0.71
  SM: 0.29
status: uninstantiated   # uninstantiated / instantiated / named
ch_id: null              # 소환 후 CH-* 발급 시 기재
```

`status` 값:
- `uninstantiated`: 좌표만 존재, 미소환
- `instantiated`: 시뮬에서 소환·조형됨, 이름 있음
- `named`: 레귤러 선정 완료, 풀 카드 보유

## 6. 실행 순서

```
[Step 1] ID 생성
  P-0001 ~ P-3600 슬롯 파일 생성 (빈 템플릿)

[Step 2] 빅5 배정
  스크립트: 정규분포 샘플링 → 3600행 CSV 생성
  → 각 슬롯 파일에 b5 섹션 기입

[Step 3] 파생 지표 산출
  배치 LLM 호출 (병렬):
    입력: b5 수치
    출력: DT / NFC / SM 수치
  → 각 슬롯 파일에 derived 섹션 기입

[Step 4] 인구 문법 문서 연결
  population_grammar.md 에 분포 요약 통계 기재
  → 오케스트레이터의 반응자 추론 시 참조 기준
```

Step 2는 파이썬 스크립트로 처리. Step 3은 Batch API 활용 시 비용 50% 절감.

---

## 7. 오케스트레이터 활용 방식

시뮬레이션 시드 수신 후:

1. 시드 분석 → 반응 예상 심리 프로파일 추론
   예) "봉문 이슈 → 높은 DT + 낮은 SM 인물이 공격적 반응"
2. population_grammar 참조 → 해당 프로파일 밀도 확인
3. character_index에서 미점유 좌표 탐색
4. 해당 좌표 슬롯(P-*)에서 캐릭터 즉석 조형
5. 시뮬 종료 후: 흥미로운 반응 → status: instantiated 또는 named로 승격
